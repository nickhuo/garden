## The Ralph loop (technique)

1. Allocate an array with the required backing specifications.
2. Assign it a goal.
3. Loop the goal repeatedly.
4. Watch for failure domains during execution.
5. Apply engineering fixes so issues never recur.
6. One task per loop iteration.
7. Run manually (prompt + CTRL+C pauses) or fully automated.
8. Relies on context engineering with LLM tokens.
9. Enables autonomous operation while the developer is away.

Treats LLMs as "a new form of programmable computer" rather than as a tool for accelerating brick-by-brick development. Software is "clay on the pottery wheel" — shaped iteratively, not stacked vertically (he derides "Jenga stacks").

## Monolithic vs multi-agent

Critiques SFO-era focus on multi-agent communication / multiplexing as premature: multi-agent systems create excessive complexity because agents are non-deterministic. Ralph is deliberately monolithic — a single OS process scaling vertically in a single repository, performing one task per loop. Avoids microservices-level complications (agent non-determinism, communication overhead).
