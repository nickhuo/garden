## 本日 research 风向 (TL;DR)

Three dominant threads today, all squarely in Nick's lane:

1. **Agents are shifting from "use given skills" → "discover + evolve skills autonomously."** GenericAgent, SkillFlow, Agent-World, EvoMaster, ClawEnvKit, and the "self-evolution via world knowledge" paper all attack the same gap: today's agents are amnesiac across episodes and rely on external reward. The new thesis: agents should grow their own library of SOPs, environments, and training tasks. **GenericAgent cracked 5.5k GitHub stars**, EvoMaster 119, OpenGame 140 — OSS momentum is huge.
2. **Eval is moving from "can you solve it?" → "are you debugging or regenerating? are you reliable? can you recognize the problem?"** Terminal Wrench (reward hacking dataset), Precise Debugging Benchmark (edit precision < 45% even at 76% pass rate), Reliability of Computer Use Agents (same task, same model, different outcome), KWBench (can LLMs even recognize the problem unprompted? 27.9% best), Agents Explore but Ignore (agents see the answer and don't use it in 50%+ of trials).
3. **Inference + post-training economics are the other half of AI Eng.** River-LLM (1.7–2.2× speedup via KV-shared early exit, training-free), Forge-UGC (6.9× faster compile, 18–35% lower latency on NPU), Pico LoRA merging (+3.4–8.3 pts), CaOPD (distillation without overconfidence trap), GFT (a cleaner unification of SFT and RL).

Bonus theme: **voice agents are catching up**. VoxMind goes 34.88% → 74.57% task completion, beating Gemini-2.5-Pro on spoken agent tasks.

## Cluster 1 — Agent frameworks & self-evolution

### [2604.17091] 🟢 GenericAgent — token-efficient self-evolving agent (5509⭐)
- **Problem:** Long-horizon LLM agents degrade because context fills with tool descriptions and memories, crowding out decision-relevant info.
- **Method:** Single principle — *context information density maximization*: atomic tool set, hierarchical on-demand memory, verified-trajectory → SOP/code self-evolution, plus truncation/compression layer.
- **Key result:** Beats leading agent systems with significantly fewer tokens and interactions; continues to evolve across episodes.

### [2604.17308] 🟢 SkillFlow — lifelong skill discovery benchmark
- **Problem:** Benchmarks test agents *using* given skills but not *discovering, repairing, maintaining* a skill library over time.
- **Method:** 166 tasks in 20 families with a Domain-Agnostic Execution Flow; agents start skill-less, externalize lessons as rubric-driven skill patches.
- **Key result:** Claude Opus 4.6 climbs 62.65% → 71.08% with lifelong evolution; but high skill usage ≠ utility (Qwen-Coder-Next regresses vs. vanilla).

### [2604.18292] 🟢 Agent-World — self-evolving training arena
- **Problem:** Training general agents is bottlenecked by lack of realistic environments and principled life-long learning.
- **Method:** Auto environment-task discovery from thousands of MCP-compatible tool ecosystems + continuous RL that dynamically synthesizes tasks to target agent weaknesses.
- **Key result:** Agent-World-8B and 14B outperform strong proprietary models on 23 agent benchmarks.

### [2604.17406] 🟢 EvoMaster — foundational evolving scientific agent (119⭐)
- **Problem:** Existing scientific-agent frameworks are static, narrow, cannot learn from trial and error.
- **Method:** Domain-agnostic base harness with continuous self-critique + hypothesis refinement; ~100 LOC to build a domain-specialized scientific agent.
- **Key result:** SOTA on Humanity's Last Exam (41.1%), MLE-Bench Lite (75.8%), BrowseComp (73.3%), FrontierScience (53.3%) — +159% to +316% over OpenClaw baseline.

### [2604.18131] 🟢 Reward-free self-evolution via world knowledge
- **Problem:** "Self-evolution" today still depends on human-defined rewards; stop supervising → evolution stops.
- **Method:** Outcome-based reward during training only measures how much the agent's self-generated world knowledge improves downstream success; at inference, no external reward needed.
- **Key result:** +20% on WebVoyager / WebWalker for Qwen3-30B & Seed-OSS-36B; a 14B Qwen3 beats unassisted Gemini-2.5-Flash.

### [2604.18543] 🟢 ClawEnvKit — auto environment generation for claw-like agents (8⭐)
- **Problem:** Constructing environments for claw-like (tool-grabbing) agents is manual, does not scale.
- **Method:** NL-description → parser → generator (task spec + tool interface + scoring) → validator pipeline; produces Auto-ClawEval, 1,040 envs × 24 categories.
- **Key result:** Matches human-curated envs at 13,800× lower cost; harness engineering adds up to 15.7 pts over bare ReAct.

### [2604.18394] 🟢 OpenGame — open agentic coding for games (140⭐)
- **Problem:** Code LLMs solve isolated programming tasks but collapse when building full playable games (cross-file state, scene wiring).
- **Method:** Game Skill (template skill + debug skill — reusable, evolving) + GameCoder-27B (continual pretrain → SFT → execution-grounded RL); OpenGame-Bench uses headless browser + VLM judging.
- **Key result:** New SOTA on 150 diverse game prompts, judged on Build Health, Visual Usability, Intent Alignment.

### [2604.17696] 🟢 Stratagem — transferable reasoning via game self-play
- **Problem:** Self-play teaches game-specific heuristics, not transferable reasoning.
- **Method:** Selectively reinforces trajectories with abstract/domain-agnostic reasoning via a Reasoning Transferability Coefficient + Reasoning Evolution Reward.
- **Key result:** Substantial gains on math, general reasoning, and code — strongest on competition-level math.

### [2604.15710] 🟢 VoxMind — agentic spoken dialogue (23⭐)
- **Problem:** End-to-end voice models lack tool-use / agentic capabilities and hit latency walls as toolset grows.
- **Method:** 470h AgentChat dataset, "Think-before-Speak" reasoning, Multi-Agent Dynamic Tool Management that async-delegates retrieval to an auxiliary agent.
- **Key result:** Task completion **34.88% → 74.57%**, beats Gemini-2.5-Pro on spoken agent tasks.

### [2604.17273] 🟡 The Continuity Layer — position paper
- **Problem:** Intelligence is powerful per session, amnesiac across time; flat memory APIs force reinterpretation every read.
- **Method:** Proposes a *continuity layer* — 7 required properties — with Decomposed Trace Convergence Memory as the storage primitive.
- **Key result:** Position paper; formal eval via companion ATANT benchmark (arXiv 2604.06710).

## Cluster 2 — Agent reliability, evaluation & safety

### [2604.17596] 🟢 Terminal Wrench — reward-hacking dataset (19⭐)
- **Problem:** Open terminal-agent benchmarks are reward-hackable; we don't have the exploit trajectories to study this.
- **Method:** 331 env subset from popular benchmarks, 3,632 hack trajectories + 2,352 legit baselines from Claude Opus 4.6, Gemini 3.1 Pro, GPT-5.4.
- **Key result:** Stripping chain-of-thought drops an LLM-judge's hack-detection AUC from **0.97 → 0.92**; exploits are task-specific, hard to patch.

### [2604.17849] 🟢 Reliability of Computer Use Agents
- **Problem:** Same task, same model, an agent succeeds once and fails on repeat — what causes this variance?
- **Method:** Repeated executions on OSWorld, paired statistical tests decomposing stochasticity, task ambiguity, behavioral variability.
- **Key result:** Reliability depends on both task specification *and* behavioral variance; authors argue evals must be *repeated* and specs must be *disambiguable via interaction*.

### [2604.17609] 🟢 Agents Explore but Agents Ignore
- **Problem:** Claim: LLM agents exploit unexpected-but-relevant environmental observations. Reality: they don't.
- **Method:** Inject full task solutions into Terminal-Bench / SWE-Bench / AppWorld environments; measure discover vs. exploit rates.
- **Key result:** Agents *see* solutions in 79–81% of Terminal-Bench runs but *use* them in 37–50%; AppWorld sees the answer in 90%+ but exploits in <7%. Three levers: tool scaffold, test-time compute, training data distribution.

### [2604.15760] 🟢 KWBench — unprompted problem recognition
- **Problem:** Existing benches test task execution, not whether the model *recognizes* the governing structure of a knowledge-work situation.
- **Method:** 223 practitioner tasks with formal game-theoretic ground truth (principal-agent, signaling, mechanism design failure, etc.); mandatory-criteria gating.
- **Key result:** Best model passes **27.9%**. Top 2 models agree on only 31.7% of passes. Routing top-8 covers 50.7% — nearly double best-single-model.

### [2604.17338] 🟢 Precise Debugging Benchmark — debug or regenerate?
- **Problem:** Frontier LLMs "debug" by regenerating whole solutions; we lack a benchmark that penalizes over-editing.
- **Method:** PDB framework — synthesize verified atomic bugs, compose into multi-bug programs, score by edit-level precision + bug-level recall.
- **Key result:** GPT-5.1-Codex / DeepSeek-V3.2-Thinking hit **>76% unit-test pass**, but **precision < 45%**; iterative/agentic strategies don't fix this.

### [2604.18224] 🟢 WebCompass — multimodal web coding eval
- **Problem:** Web-coding benchmarks test narrow slices (text → code, static metrics); miss visual fidelity, interaction quality, codebase-level reasoning.
- **Method:** 3 modalities (text/image/video) × 3 task types (gen/edit/repair) = 7 categories; novel Agent-as-a-Judge executes generated sites in a real browser via MCP.
- **Key result:** Closed models clearly ahead; repair preserves interactivity better than editing; aesthetics is the biggest open-source gap; Vue > Vanilla/React in difficulty.

### [2604.15579] 🟢 Symbolic Guardrails — when to skip neural guardrails (1⭐)
- **Problem:** Agent safety leans on training + neural guardrails, which give probabilistic not provable guarantees.
- **Method:** Systematic review of 80 safety/security benchmarks → map policy requirements → evaluate symbolic guardrails on τ²-Bench, CAR-bench, MedAgentBench.
- **Key result:** 85% of benchmarks lack concrete policies. Of specified ones, **74% are enforceable by symbolic rules** without sacrificing utility.

### [2604.17698] 🟡 Geometric Canary — representation stability diagnostic
- **Problem:** Predicting *steerability* pre-deployment and detecting *drift* post-deployment feel like different problems.
- **Method:** Both reduce to geometric stability; supervised Shesha for steerability, unsupervised for drift.
- **Key result:** Steerability prediction ρ=0.89–0.97; drift detection 2× CKA signal, up to 5.23× on Llama, 73% earlier warning, 6× lower false-alarm vs Procrustes.

### [2511.10262] 🟢 MTR-DuplexBench — multi-round full-duplex speech eval (3⭐)
- **Problem:** Full-duplex speech LMs benchmarked only on single-round; multi-turn (turn boundaries, context consistency) untested.
- **Method:** Segment continuous duplex dialogues into turn units; evaluate conversational features, quality, instruction-following, safety.
- **Key result:** FD-SLMs struggle to maintain consistent performance across rounds and dimensions; benchmark released.

### [2604.17886] 🟢 Latent Preference Modeling — cross-session tool calling
- **Problem:** Users omit details → under-specified tool arguments; agents must infer preferences across sessions.
- **Method:** MPT benchmark (265 multi-session dialogues); PRefine — test-time memory as evolving hypotheses, generate→verify→refine loop.
- **Key result:** Improves tool-call accuracy while using **1.24% of the tokens** full-history prompting would.

### [2604.16593] ⚪ SemanticQA — semantic phrase benchmark (6⭐)
- **Problem:** No unified testbed for multiword expressions (idioms, noun compounds, verbal constructions) across LM architectures.
- **Method:** Consolidates existing MwE resources into extraction / classification / interpretation tasks + sequential compositions.
- **Key result:** Large performance variance on semantic-reasoning subtasks; insight target: non-trivial phrase comprehension.

## Cluster 3 — Post-training: RL, distillation, merging

### [2604.14258] 🟢 GFT — Group Fine-Tuning (25⭐)
- **Problem:** SFT is a degenerate case of policy gradient with sparse implicit reward + unstable inverse-prob weighting → entropy collapse, gradient explosion.
- **Method:** Group Advantage Learning (contrastive normalized supervision over response groups) + Dynamic Coefficient Rectification (bounds IPW).
- **Key result:** Beats SFT-style methods consistently; integrates more smoothly with downstream RL.

### [2604.16830] 🟢 CaOPD — calibration-aware on-policy distillation (1⭐)
- **Problem:** On-policy distillation (OPD) trains an information mismatch: teacher supervises with privileged context → severe overconfidence (a "Scaling Law of Miscalibration").
- **Method:** Replace teacher-conditioned success targets with empirical confidence from student rollouts; distill the revised responses through self-distillation.
- **Key result:** Pareto-optimal calibration with competitive capability; generalizes OOD and to continual learning.

### [2604.16826] 🟢 Pico — LoRA merge calibration
- **Problem:** Merging independently-trained LoRAs usually loses performance; existing methods treat B×A as one object.
- **Method:** Observation — B repeatedly uses a small set of shared directions across tasks, A is task-specific; Pico downscales over-shared directions in B pre-merge, rescales after.
- **Key result:** **+3.4–8.3 avg accuracy points** on 8 benchmarks (math/code/finance/medical); merged adapter even beats a single LoRA trained on all task data.

### [2604.18574] 🟢 RLVR under weak supervision (2⭐)
- **Problem:** When can RL with verifiable rewards generalize under scarce data, noisy rewards, or self-supervised proxies?
- **Method:** Empirical study across model families; key predictor is *reasoning faithfulness* (do intermediate steps support the answer?), not output diversity.
- **Key result:** Generalizing models show a *prolonged pre-saturation phase*; SFT on explicit reasoning traces is necessary, and continual pre-training amplifies it.

### [2602.15143] 🟡 Trace Rewriting — anti-distillation + watermarking (1⭐)
- **Problem:** Competitors distill frontier models; teachers have no watermark or defense.
- **Method:** Rewrite teacher reasoning traces (instruction-based LLM rewriting / gradient-based) preserving answer correctness but degrading student training signal; optionally embed detectable watermark.
- **Key result:** Strong anti-distillation effect while maintaining teacher performance; watermark detection with essentially zero false alarms.

### [2604.16893] 🟢 EasyVideoR1 — RL framework for video understanding (50⭐)
- **Problem:** Video RL suffers from high-dim preprocessing overhead, task-type diversity, hyperparameter sensitivity; open frameworks lack video-specific optimizations.
- **Method:** Offline preprocessing + tensor caching (1.47× throughput); 11-task-type reward routing; mixed offline-online data; joint image-video training; async multi-benchmark eval (22 benchmarks).
- **Key result:** Full video-RL pipeline released with reproducibility claims close to reported scores.

### [2604.17972] ⚪ Multi-strategy ESC
- **Problem:** Emotional support conversation models assume one strategy per turn; real supporters blend multiple strategies per utterance.
- **Method:** All-in-One (single decode) and One-by-One (iterative) decoders, both enhanced with RL-guided cognitive reasoning.
- **Key result:** Improved supportive quality and dialogue success on ESConv; first empirical evidence that multi-strategy per turn is beneficial.

## Cluster 4 — Inference optimization & compiler

### [2604.18396] 🟢 River-LLM — training-free KV-shared early exit
- **Problem:** Early-exit inference is bottlenecked by KV Cache Absence — skipped layers leave holes in historical KV; recomputation or masking is lossy or slow.
- **Method:** KV-Shared Exit River: backbone's missing KV generated/preserved naturally during exit; state-transition-similarity signal guides exit decisions.
- **Key result:** **1.71–2.16× wall-clock speedup** with quality preserved on math reasoning + code gen.

### [2604.16498] 🟢 Forge-UGC — four-phase NPU transformer compiler
- **Problem:** OpenVINO / ONNX Runtime have opaque passes, weak buffer management; high compile cost and runtime overhead on NPU.
- **Method:** (1) torch.export at ATen level, (2) six FX optimization passes (CSE, fusion, layout, etc.) cutting graph nodes 14–22%, (3) typed IR with virtual registers, (4) liveness-analysis linear-scan allocation + device-affinity scheduling.
- **Key result:** **6.9–9.2× faster compile**, **18–36% lower latency**, **30–41% lower energy** on 125M–8B models; fidelity preserved (max logit diff < 2.1e-5).

## Cluster 5 — Retrieval / RAG

### [2604.16576] 🟢 Robustness of LLM-based dense retrievers
- **Problem:** Decoder-only LLMs now replace BERT as retrieval backbones but their robustness is unstudied.
- **Method:** 30-dataset eval across 4 benchmarks; linear mixed-effects disentangle model-vs-dataset variance; tests vs. typos, paraphrasing, adversarial corpus poisoning.
- **Key result:** Reasoning-optimized models pay a "specialization tax" — worse generalization. LLM retrievers survive typos and corpus poisoning better than encoders but remain vulnerable to *semantic* perturbations (synonymizing). Embedding angular uniformity predicts lexical stability; scale helps robustness.

## Cluster 6 — Multimodal, world models, generation

### [2604.18486] 🟢 OneVL — latent CoT for autonomous driving
- **Problem:** Latent CoT methods compress reasoning to hidden states but lose to explicit CoT because they compress symbolic not causal-dynamics info.
- **Method:** Dual decoders supervise the latent space — language decoder rebuilds text CoT, *visual world model decoder* predicts future-frame tokens; three-stage training aligns latents with trajectory, language, visual objectives.
- **Key result:** First latent-CoT method to **surpass explicit CoT** across 4 benchmarks at answer-only latency.

### [2604.18564] 🟢 MultiWorld — multi-agent multi-view video world model (77⭐)
- **Problem:** Existing world models are single-agent; can't capture multi-agent interactions or multi-view consistency.
- **Method:** Multi-Agent Condition Module + Global State Encoder; synthesizes different views in parallel.
- **Key result:** Beats baselines in video fidelity, action-following, multi-view consistency on multi-player games + multi-robot manip.

### [2604.16902] 🟡 Beyond Text-Dominance — modality preference (5⭐)
- **Problem:** Native omni-modal LLMs unify modalities — what's their *preference* under cross-modal conflict?
- **Method:** Conflict-based benchmark + modality selection rate; layer-wise probing reveals preference emerges mid-to-late layers.
- **Key result:** OLLMs are *visually dominant* (reversing the old text-dominance of VLMs); internal preference signals enable competitive cross-modal hallucination diagnosis without task-specific data.

### [2604.16943] 🟡 MNAFT — modality neuron-aware fine-tuning
- **Problem:** Full or PEFT fine-tuning for image translation introduces parameter redundancy; vision-text modality gap persists.
- **Method:** Instruction-driven activation analysis identifies language-specific / language-agnostic neurons across vision + language modules; fine-tune only those.
- **Key result:** Beats cascaded, full-FT, and PEFT baselines on image-translation benchmarks.

### [2604.18168] 🟡 MeanFlow → text-conditioned (85⭐)
- **Problem:** MeanFlow one-step generation works for class labels; text conditions fail with naive LLM-encoder injection.
- **Method:** Shows few-step refinement demands *highly discriminable* text features; uses an LLM text encoder validated to have that property and adapts the MeanFlow pipeline.
- **Key result:** First efficient one-step text-conditioned MeanFlow synthesis; validated gains on standard diffusion too.

### [2604.18203] 🟡 Multimodal multiplication failure modes
- **Problem:** MLLMs perceive numbers cross-modally but fail at exact multi-digit multiplication regardless of modality.
- **Method:** Factorial benchmark over digit length × sparsity × representation × modality + "arithmetic load" C = total × non-zero digit count; perception-vs-computation probes.
- **Key result:** Accuracy collapses as C grows (near-zero at C>100); failures are *computational* not perceptual (>99% on matched-perception); models default to decomposition heuristics.

### [2604.11102] ⚪ OmniScript — video-to-script (8B)
- **Problem:** Long-form cinematic video → temporally grounded script is beyond current MLLMs.
- **Method:** 8B audio-visual model; CoT-SFT for plot/character → RL with temporally segmented rewards; hierarchical eval framework.
- **Key result:** Matches proprietary SOTA (incl. Gemini 3-Pro) at 8B.

### [2604.13313] ⚪ Concrete Jungle — compositional VLM training
- **Problem:** VLMs struggle with compositional reasoning; hard-negative mining lacks explicit control over which tokens to modify.
- **Method:** ConcretePlant (manipulates highly concrete terms for stronger signal) + Cement loss (margin-based, psycholinguistics-calibrated); framework named Slipform.
- **Key result:** SOTA on compositional benchmarks + retrieval + linear probing.

### [2604.18267] ⚪ MARCO — semantic correspondence (25⭐)
- **Problem:** Dual-encoder (DINOv2 + diffusion) correspondence models are huge and generalize poorly beyond trained keypoints.
- **Method:** Coarse-to-fine objective + self-distillation turns sparse keypoints into dense semantically coherent correspondence; pure DINOv2 backbone.
- **Key result:** SOTA on SPair-71k, AP-10K, PF-PASCAL; +8.9 PCK@0.01, +5.1 on unseen keypoints, +4.7 on unseen categories; 3× smaller, 10× faster than diffusion approaches.

### [2604.17454] ⚪ Hyperbolic Scene Graph (1⭐)
- **Problem:** Scene graphs embedded in Euclidean space don't naturally capture hierarchical place/object entailment.
- **Method:** Learn scene graph embeddings in hyperbolic space.
- **Key result:** Graph-level IoU 33.51 vs. 25.37 best baseline (+8.14).

## Cluster 7 — Domain-specific (bio, med, cyber, time series)

### [2604.16038] 🟡 Vulnerability sightings forecasting (3⭐)
- **Problem:** PoC releases, detection templates, online chatter are sparse and bursty → standard forecasters fail.
- **Method:** SARIMAX with VLAI-severity covariates vs. Poisson count regression vs. exponential-decay baselines.
- **Key result:** Poisson weekly aggregates give more stable forecasts than SARIMAX (which produces negative values / wide CIs on short bursty data).

### [2604.06505] ⚪ MedConclusion — biomedical conclusion generation (1⭐)
- **Problem:** No large naturally-occurring dataset for testing if LLMs can infer conclusions from structured biomedical evidence.
- **Method:** 5.7M PubMed abstract-to-conclusion pairs with journal/SJR metadata; eval under conclusion vs. summary prompting with LLM-as-judge.
- **Key result:** Conclusion writing is behaviorally distinct from summary writing; strong models cluster under current metrics; judge identity materially shifts absolute scores.

### [2604.17318] ⚪ MedFocusLeak — transferable attack on medical VLMs (2⭐)
- **Problem:** Medical VLMs being deployed in diagnostics with untested adversarial robustness.
- **Method:** Black-box transferable attack injecting imperceptible perturbations into non-diagnostic background; attention-distraction away from pathological regions.
- **Key result:** SOTA attack success across six medical modalities producing clinically plausible but wrong diagnoses; new unified fidelity+success evaluation framework.

### [2604.08537] ⚪ Meta-learned in-context brain decoding (10⭐)
- **Problem:** Cross-subject fMRI decoders need bespoke per-subject training.
- **Method:** Meta-optimized in-context model: condition on few image-brain examples; hierarchical inference inverts the encoder at voxel + region level.
- **Key result:** Cross-subject + cross-scanner generalization without retraining, fine-tuning, anatomical alignment, or stimulus overlap.

### [2604.03337] ⚪ RGxEStat — GxE interaction tool (0⭐)
- **Problem:** Breeders/agronomists need GxE stats without SAS/R programming overhead.
- **Method:** Significance analysis (mixed-effects) + stability analysis packaged with a GUI.
- **Key result:** Tool released; no headline number reported.

### [2604.16642] ⚪ Shesha — single-cell CRISPR geometry
- **Problem:** Perturbation "effect magnitude" ignores whether cells move *together*; same magnitude can mean coherent or scattered response.
- **Method:** Directional coherence metric (mean cosine sim of per-cell shifts to mean perturbation direction) over 2,200+ perturbations.
- **Key result:** Stability correlates 0.75–0.97 with magnitude; discordant cases reveal regulatory architecture (master regulators pay "geometric tax"); low stability correlates with chaperone stress activation.

### [2604.17388] ⚪ JuRe — minimal denoising net for time-series anomaly
- **Problem:** Time-series AD architectures have been getting more complex (attention, latent vars, adversarial) without clear ROI.
- **Method:** Depthwise-separable 128-dim residual block trained to repair corrupted windows; parameter-free structural discrepancy score at inference.
- **Key result:** #2 on TSB-AD multivariate (AUC-PR 0.404) and UCR univariate (0.198); training-time corruption is the dominant factor (ΔAUC-PR 0.047), not capacity.

### [2604.18584] 🟡 MathNet — multilingual Olympiad math benchmark (1⭐)
- **Problem:** Math benchmarks are narrow, mono-lingual, lack retrieval task.
- **Method:** 30,676 expert problems from 47 countries × 17 languages × 20 years; three tasks (solve / math-aware retrieval / RAG solve).
- **Key result:** Gemini-3.1-Pro 78.4%, GPT-5 69.3%; RAG performance is highly sensitive to retrieval quality (DeepSeek-V3.2-Speciale gains up to +12% with good retrieval).
