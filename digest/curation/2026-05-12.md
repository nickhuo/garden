## Part 2 — Research Radar (HuggingFace Papers, English)

### TL;DR — Today's Research Wind

Three overlapping threads dominate today's 51 papers: **(1) agent skill lifecycle** — how to dynamically grow, prune, and govern what an agent can do (SLIM, FORTIS, NanoResearch, FlashEvolve); **(2) self-evolution without human labels** — RL methods that bootstrap from self-generated signals (SEIF, G-Zero, Rebellious Student, Auto-Rubric); **(3) inference efficiency** — KV cache eviction, speculative decoding, MoE compression for edge deployment (Make Each Token Count, SlimSpec, DECO). The agent skill papers are unusually actionable for Donut Labs architecture thinking.

### Agent frameworks & self-evolution

#### 🟢 [2605.10923] Dynamic Skill Lifecycle Management for Agentic RL (SLIM)
- **Problem:** Agent skill sets typically only grow, but not all skills remain relevant — marginal contribution is uneven and task-dependent.
- **Method:** SLIM framework for non-monotonic skill lifecycle management: selective activation, dormancy, and pruning based on task/stage context.
- **Key result:** No quantitative result reported in abstract; framework paper.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10923) · [HF](https://huggingface.co/papers/2605.10923)

#### 🟢 [2605.10813] NanoResearch: Co-Evolving Skills, Memory, Policy for Research Automation
- **Problem:** LLM research agents produce uniform outputs regardless of user's resource constraints, preferences, and target format — personalization missing.
- **Method:** Co-evolving three components simultaneously: skill library, episodic memory, and policy, all adapting to individual researcher profiles.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10813) · [HF](https://huggingface.co/papers/2605.10813)

#### 🟢 [2605.08520] FlashEvolve: Accelerating Agent Self-Evolution with Async Orchestration
- **Problem:** LLM-based agent evolution (refining non-parametric artifacts) is bottlenecked by synchronized stage execution and intra-stage imbalance.
- **Method:** Asynchronous workers + queues allow stages to overlap; versioned artifact tracking handles staleness with update/discard/patch policies.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.08520) · [HF](https://huggingface.co/papers/2605.08520)

### Agent reliability, evaluation & safety

#### 🟢 [2605.08715] AgentForesight: Online Auditing for Early Failure Prediction in Multi-Agent Systems
- **Problem:** Errors in multi-agent long-horizon tasks cascade — current work diagnoses failure post-hoc, too late to intervene.
- **Method:** Online auditor observes unfolding trajectory step-by-step and predicts failure probability before trajectory ends, enabling early intervention.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.08715) · [HF](https://huggingface.co/papers/2605.08715)

#### 🟢 [2605.09163] FORTIS: Benchmarking Over-Privilege in Agent Skills
- **Problem:** Agent skill layers are treated as organizational abstractions, but models routinely exceed the privilege boundary of the skill they invoked.
- **Method:** Benchmark evaluating two failure modes: (1) selecting a minimal-sufficient skill from overlapping library, (2) not expanding into broader tools than the skill permits.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09163) · [HF](https://huggingface.co/papers/2605.09163)

#### 🟡 [2605.10344] TMAS: Scaling Test-Time Compute via Multi-Agent Synergy
- **Problem:** Existing structured test-time scaling either weakly coordinates parallel trajectories or uses noisy historical info without deciding what to retain.
- **Method:** Multi-agent system with explicit coordination of reasoning trajectories, retention-based memory, and verification feedback loops.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10344) · [HF](https://huggingface.co/papers/2605.10344)

### Post-training (RL, distillation, merging, calibration)

#### 🟡 [2605.07465] SEIF: Self-Evolving RL for Instruction Following
- **Problem:** Improving instruction following continuously is hard — human labels are costly, self-play with static difficulty stagnates as model improves.
- **Method:** Closed self-evolution loop where a generator model and an instruction proposer co-evolve via RL, with instructions automatically increasing in difficulty.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.07465) · [HF](https://huggingface.co/papers/2605.07465)

#### 🟡 [2605.09959] G-Zero: Self-Play for Open-Ended Generation Without Verifiers
- **Problem:** Self-evolving LLMs rely on LLM judges for open-ended tasks, introducing reward hacking and capability bottlenecks.
- **Method:** Verifier-free Hint-δ intrinsic reward measures predictive shift between unassisted and hint-conditioned responses; Proposer trained via GRPO to target Generator's blind spots.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09959) · [HF](https://huggingface.co/papers/2605.09959)

#### 🟡 [2605.10781] Rebellious Student: Reversing Teacher Signals for Reasoning Exploration
- **Problem:** Self-distillation overwrites the student's successful non-teacher-predicted reasoning, suppressing independent exploration.
- **Method:** Inverts the distillation signal on successful student rollouts the teacher wouldn't have predicted, reinforcing self-driven reasoning paths.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10781) · [HF](https://huggingface.co/papers/2605.10781)

### Inference optimization & compiler

#### 🟢 [2605.09649] Make Each Token Count: KV Cache Eviction for Long-Context Performance
- **Problem:** KV cache is the main bottleneck in long-context inference; existing eviction methods degrade performance vs full cache.
- **Method:** Global retention-based eviction that learns each token's future utility rather than using heuristics; argues selective eviction can *improve* over full-cache attention by removing distracting tokens.
- **Key result:** No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09649) · [HF](https://huggingface.co/papers/2605.09649)

#### 🟡 [2605.10933] DECO: Sparse MoE with Dense-Comparable Performance for Edge Devices
- **Problem:** MoE scales capacity but its total parameter footprint creates memory-access bottlenecks on end-side devices.
- **Method:** Sparse MoE with differentiable ReLU-based routing under identical total-parameter and training-token budget as dense Transformer.
- **Key result:** Dense-comparable performance claimed; no specific metric in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10933) · [HF](https://huggingface.co/papers/2605.10933)
