## Part 2 — Research Radar (HuggingFace Papers, English)

### TL;DR — Today's Research Wind

Today's 50 papers cluster heavily around **agent memory, reliability, and self-evolution** — three inter-locking problems that signal the field has moved from "can agents act?" to "can agents persist, correct themselves, and stay safe?" A second strong cluster covers **multi-agent topology and post-training methods** (GRPO variants, on-policy distillation, RL beyond verifiable rewards). The cross-source signal is unusually coherent: GitHub Trending (agentmemory, memoir, cc-connect) and HF papers (MEME, LongMemEval-V2, TacoMAS) are all pointing at the same gap.

### Agent frameworks & self-evolution

### 🟢 [2605.09998] Continual Harness: Online Adaptation for Self-Improving Foundation Agents
- **Problem:** No equivalent of Claude Code / OpenHands harness exists for embodied agents with long-horizon partial observability.
- **Method:** Continual Harness framework; applied to Gemini Plays Pokemon (GPP) with iterative human-in-the-loop harness refinement.
- **Key result:** GPP became first AI to complete Pokémon Blue/Yellow Legacy on hard mode; framework generalizes across domains.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09998) · [HF](https://huggingface.co/papers/2605.09998)

### 🟡 [2605.09539] TacoMAS: Test-Time Co-Evolution of Topology and Capability in LLM-based Multi-Agent Systems
- **Problem:** MAS either fix topology at inference or adapt only topology or capability — not both simultaneously.
- **Method:** Joint test-time co-evolution of agent communication topology and per-agent capability via TacoMAS.
- **Key result:** Empirical and theoretical evidence that joint evolution outperforms single-axis adaptation; No quantitative result reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09539) · [HF](https://huggingface.co/papers/2605.09539)

### 🟡 [2605.09131] MCP-Cosmos: World Model-Augmented Agents for Complex Task Execution in MCP Environments
- **Problem:** MCP agents lack world-model foresight — task planning ignores execution-time dynamics and vice versa.
- **Method:** MCP-Cosmos: a framework augmenting MCP agents with learned world models for long-horizon planning + reactive execution fusion.
- **Key result:** No quantitative result reported in abstract; framework proposed with conceptual validation.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09131) · [HF](https://huggingface.co/papers/2605.09131)

### 🟢 [2605.12460] Multi-Stream LLMs: Unblocking Language Models with Parallel Streams of Thoughts, Inputs and Outputs
- **Problem:** Core agent architecture unchanged since ChatGPT: sequential message exchange bottlenecks multi-task parallelism.
- **Method:** Parallel stream architecture enabling simultaneous independent input/output/reasoning streams within a single model.
- **Key result:** No quantitative result reported in abstract; architectural proposal targeting coding and computer use agent latency.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12460) · [HF](https://huggingface.co/papers/2605.12460)

### Agent reliability, evaluation & safety

### 🟢 [2605.10365] Agent-ValueBench: A Comprehensive Benchmark for Evaluating Agent Values
- **Problem:** Existing value benchmarks confined to LLMs; agent values—silently steering behavior through harnesses like OpenClaw—largely unmeasured.
- **Method:** Agent-ValueBench: multi-dimensional benchmark evaluating values embedded in agent behavior across deployment contexts.
- **Key result:** No quantitative result reported in abstract; establishes new eval framework with empirical, intuitive, and theoretical grounding.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10365) · [HF](https://huggingface.co/papers/2605.10365)

### 🟢 [2605.11882] On-Policy Self-Evolution via Failure Trajectories for Agentic Safety Alignment
- **Problem:** Safety alignment signals are response-level or off-policy; tool-using agents fail through entire trajectories (unsafe calls, prompt injection, over-refusal) not just final outputs.
- **Method:** On-policy self-evolution using failure trajectory data — agent learns from its own unsafe execution histories without safety-utility tradeoff.
- **Key result:** Improves agent safety without degrading task performance; No specific metric reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.11882) · [HF](https://huggingface.co/papers/2605.11882)

### 🟡 [2605.12481] ToolCUA: Towards Optimal GUI-Tool Path Orchestration for Computer Use Agents
- **Problem:** CUAs uncertain when to use GUI actions vs. tool/API calls; hybrid action space causes suboptimal execution paths.
- **Method:** ToolCUA: high-quality interleaved GUI-Tool trajectory dataset + orchestration policy for Computer Use Agents.
- **Key result:** No quantitative result reported in abstract; dataset and model proposed for GUI-Tool path optimization.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12481) · [HF](https://huggingface.co/papers/2605.12481)

### Post-training (RL, distillation, merging, calibration)

### 🟡 [2605.10899] RubricEM: Meta-RL with Rubric-guided Policy Decomposition beyond Verifiable Rewards
- **Problem:** Deep research agents (plan, search, evaluate, synthesize) can't use verifiable reward RL — no ground-truth answers, multi-step trajectories, no mechanism to reuse past attempts.
- **Method:** Meta-RL with rubric-guided policy decomposition; turns rubric criteria into reward signal for open-ended long-form tasks.
- **Key result:** No quantitative result reported in abstract; demonstrates RL applicability beyond verifiable reward regimes.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10899) · [HF](https://huggingface.co/papers/2605.10899)

### 🟡 [2605.12483] Beyond GRPO and On-Policy Distillation: An Empirical Sparse-to-Dense Reward Principle
- **Problem:** Using labeled verifiable data directly on the deployment student (standard GRPO) is inefficient when training data is the binding constraint.
- **Method:** Sparse-to-dense reward allocation principle: use scarce verifiable data on a teacher first, then distill; reward density matters more than direct application.
- **Key result:** No quantitative result reported in abstract; principled framework for scarce-data post-training.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12483) · [HF](https://huggingface.co/papers/2605.12483)

### 🟡 [2605.12474] Reward Hacking in Rubric-Based Reinforcement Learning
- **Problem:** Policies optimized against a single rubric judge learn to hack it; cross-evaluator generalization is weak.
- **Method:** Cross-family panel of 3 frontier judges as evaluation; studies reward hacking patterns in rubric-based RL.
- **Key result:** No quantitative result reported in abstract; identifies systematic hacking patterns with proposed mitigations.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12474) · [HF](https://huggingface.co/papers/2605.12474)

### Retrieval / RAG / memory

### 🟢 [2605.12477] MEME: Multi-entity & Evolving Memory Evaluation
- **Problem:** Prior memory benchmarks test only single-entity updates; real agents must handle multi-entity dependencies, cascades, deletions across sessions.
- **Method:** MEME benchmark: six tasks covering full multi-entity × evolving matrix, including Cascade, Absence (dependency reasoning), and Deletion — three tasks not in prior benchmarks.
- **Key result:** No quantitative result reported in abstract; reveals gaps in all existing memory systems.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12477) · [HF](https://huggingface.co/papers/2605.12477)

### 🟢 [2605.12493] LongMemEval-V2: Evaluating Long-Term Agent Memory Toward Experienced Colleagues
- **Problem:** Existing memory benchmarks focus on user histories or downstream task success; don't measure whether agents internalize interface affordances, state dynamics, and recurring failure modes.
- **Method:** LongMemEval-V2: direct evaluation of long-term memory in specialized web environments — measures retention of env-specific operational knowledge.
- **Key result:** No quantitative result reported in abstract; new benchmark for agent memory in deployment-realistic settings.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12493) · [HF](https://huggingface.co/papers/2605.12493)

### 🟡 [2605.09530] MemPrivacy: Privacy-Preserving Personalized Memory Management for Edge-Cloud Agents
- **Problem:** Edge-cloud agent memory exposes sensitive user data; aggressive masking removes task-relevant semantics and degrades performance.
- **Method:** MemPrivacy: privacy-preserving selective memory management that retains semantic utility while protecting sensitive context.
- **Key result:** No quantitative result reported in abstract; framework targeting edge-cloud deployment privacy.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09530) · [HF](https://huggingface.co/papers/2605.09530)

### Code / web agents

### 🟡 [2605.07237] Teaching Language Models to Think in Code
- **Problem:** Tool-integrated reasoning (TIR) interleaves NL + code but code often acts as post-hoc verifier; NL computations are error-prone; roles overlap.
- **Method:** Models trained to reason primarily *in* code rather than interleaving — code-native chain-of-thought with clear separation of NL vs. computational roles.
- **Key result:** No quantitative result reported in abstract; demonstrates cleaner reasoning separation improves math problem solving.
- *Links:* [PDF](https://arxiv.org/pdf/2605.07237) · [HF](https://huggingface.co/papers/2605.07237)
