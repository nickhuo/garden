## Part 2 — Research Radar (HuggingFace Papers, English)

### TL;DR — Today's Research Wind

Three dominant threads today: (1) **agentic memory and on-policy training** — multiple papers attacking the problem of how agents remember, update, and learn from environment interaction; (2) **agent reliability and evaluation** — AgentLens, RealICU, and SafeHarbor all push on the question of whether our eval methods are actually measuring agent capability; (3) **inference efficiency and parallel decoding** — Orthrus and the energy-to-token position paper signal that the field is starting to treat compute budget as a first-class constraint. Strong signal day for anyone tracking agent architecture.

---

### Agent Frameworks & Self-Evolution

#### 🟢 [2605.09942] HAGE: Agentic Memory via RL-Driven Weighted Graph Evolution
- **Problem:** LLM agents lack adaptive, structured long-horizon memory across sessions.
- **Method:** Weighted multi-relational memory graphs with RL-based traversal optimization and query-conditioned retrieval.
- **Key result:** Improved long-horizon reasoning accuracy on standard benchmarks; no single quantitative figure reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09942) · [HF](https://huggingface.co/papers/2605.09942)

#### 🟢 [2605.13037] MAP: Map-then-Act Paradigm for Long-Horizon Agent Reasoning
- **Problem:** Reactive agents suffer from delayed perception and epistemic bottlenecks during long task execution.
- **Method:** Pre-execution global exploration + task-specific environment mapping + knowledge-augmented action generation.
- **Key result:** No quantitative result reported in abstract; paradigm validated on interactive agent benchmarks.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13037) · [HF](https://huggingface.co/papers/2605.13037)

#### 🟡 [2605.13481] PersonalAI 2.0: Knowledge Graph Traversal for Personalized LLM Agents
- **Problem:** Personalized agents lack dynamic, structured access to user-specific knowledge.
- **Method:** External KG integration with multistage query processing and adaptive information search.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13481) · [HF](https://huggingface.co/papers/2605.13481)

#### 🟡 [2605.11550] The DAWN of World-Action Interactive Models
- **Problem:** World models for planning fail to jointly model scene evolution and action sequences.
- **Method:** Recursive refinement over world-state and action sequences for long-horizon autonomous planning.
- **Key result:** No quantitative result reported; demonstrated on autonomous driving scenarios.
- *Links:* [PDF](https://arxiv.org/pdf/2605.11550) · [HF](https://huggingface.co/papers/2605.11550)

### Agent Reliability, Evaluation & Safety

#### 🟢 [2605.12925] AgentLens: The Lucky Pass Problem in SWE-Agent Evaluation
- **Problem:** SWE-bench evaluations mask the difference between genuinely capable agents and those that pass tests by chance ("lucky passes").
- **Method:** Process-level evaluation framework with quality scoring over agent trajectories, not just pass/fail.
- **Key result:** Reveals systematic patterns distinguishing effective vs. ineffective agent approaches; no single metric reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12925) · [HF](https://huggingface.co/papers/2605.12925)

#### 🟢 [2605.05704] SafeHarbor: Hierarchical Memory-Augmented Guardrail for LLM Agent Safety
- **Problem:** LLM agents lack precise, context-aware safety boundaries that balance utility with harm prevention.
- **Method:** Hierarchical memory system with self-evolving defense rules and context-aware policy matching.
- **Key result:** No quantitative result reported; evaluated on agent safety benchmarks.
- *Links:* [PDF](https://arxiv.org/pdf/2605.05704) · [HF](https://huggingface.co/papers/2605.05704)

#### 🟡 [2605.13542] RealICU: Do LLM Agents Understand Long-Context ICU Data?
- **Problem:** No rigorous benchmark for LLM clinical agents on long-horizon real ICU patient trajectories.
- **Method:** Hindsight-annotated patient trajectory benchmark with clinical recommendation evaluation.
- **Key result:** LLM agents show significant limitations in clinical accuracy and early interpretation bias; no exact numbers in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13542) · [HF](https://huggingface.co/papers/2605.13542)

### Post-Training (RL, Distillation, Calibration)

#### 🟢 [2605.12913] Revisiting DAgger in the Era of LLM-Agents
- **Problem:** Long-horizon LLM agents trained via SFT or RL alone miss the benefits of both; DAgger is under-explored for language agents.
- **Method:** Teacher-student policy interpolation with on-policy interactions, combining SFT warmup and RL exploration.
- **Key result:** No quantitative result reported; shown effective on long-horizon agent benchmarks.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12913) · [HF](https://huggingface.co/papers/2605.12913)

#### 🟢 [2605.12004] Learning Agentic Policy from Action Guidance
- **Problem:** RL for LLM agents is expensive; SFT requires large curated datasets; exploration is inefficient.
- **Method:** Human interaction action data as reference guidance, reducing SFT dependency and improving RL exploration.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12004) · [HF](https://huggingface.co/papers/2605.12004)

#### 🟡 [2605.08737] The Extrapolation Cliff in On-Policy Distillation of Near-Deterministic Structured Outputs
- **Problem:** On-policy distillation breaks structured output format preservation beyond a reward extrapolation threshold.
- **Method:** Empirical analysis of safety threshold; performance parity at reduced parameter count below threshold.
- **Key result:** Format preservation holds below threshold; collapses sharply above it ("extrapolation cliff").
- *Links:* [PDF](https://arxiv.org/pdf/2605.08737) · [HF](https://huggingface.co/papers/2605.08737)

#### 🟡 [2605.12995] F-GRPO: Factorized Group-Relative Policy Optimization for Unified Generation and Ranking
- **Problem:** Credit assignment in end-to-end retrieval fails when generation and ranking are trained separately.
- **Method:** Factorized GRPO combining candidate generation and ranking in a single autoregressive model.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12995) · [HF](https://huggingface.co/papers/2605.12995)

### Inference Optimization & Compiler

#### 🟢 [2605.13779] MinT: Managed Infrastructure for Training and Serving Millions of LLMs
- **Problem:** Serving millions of fine-tuned LLM adapters is prohibitively expensive; existing infra doesn't scale adapter management.
- **Method:** Base models stay resident; low-rank adapter revisions are moved dynamically across distributed policy management.
- **Key result:** Scales across large model architectures with reduced storage requirements; no exact throughput numbers in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13779) · [HF](https://huggingface.co/papers/2605.13779)

#### 🟡 [2605.12825] Orthrus: Memory-Efficient Parallel Token Generation via Dual-View Diffusion
- **Problem:** Parallel token generation (diffusion-style) sacrifices inference fidelity vs. autoregressive LLMs.
- **Method:** Dual-architecture framework pairing AR LLM with diffusion model via shared KV caches and consensus decoding.
- **Key result:** Fast parallel generation with exact inference fidelity maintained; no latency numbers reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12825) · [HF](https://huggingface.co/papers/2605.12825)

#### ⚪ [2605.11733] Position: LLM Inference Should Be Evaluated as Energy-to-Token Production
- **Problem:** Accuracy and latency metrics ignore compute, power, and cooling constraints in real deployments.
- **Method:** Position paper proposing energy-to-token as the primary inference evaluation metric.
- **Key result:** No quantitative result reported; conceptual framework only.
- *Links:* [PDF](https://arxiv.org/pdf/2605.11733) · [HF](https://huggingface.co/papers/2605.11733)

### Retrieval / RAG / Memory

#### 🟢 [2605.12975] Retrieval is Cheap, Show Me the Code: Executable Multi-Hop Reasoning for RAG
- **Problem:** Multi-hop QA over retrieved documents fails with unstructured chain-of-thought; retrieval errors compound.
- **Method:** Reframes multi-hop QA as program synthesis: write executable code over retrieved docs, run it, use deterministic feedback.
- **Key result:** Outperforms existing RAG approaches on multi-hop benchmarks; no exact numbers in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.12975) · [HF](https://huggingface.co/papers/2605.12975)

#### 🟢 [2605.10268] MemReread: Enhancing Agentic Long-Context Reasoning via Memory-Guided Rereading
- **Problem:** Long-context agents discard information early; retrieval introduces its own errors.
- **Method:** Question decomposition + selective rereading of original context (no intermediate retrieval); linear time complexity.
- **Key result:** Recovers discarded information with linear complexity; no exact benchmark numbers.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10268) · [HF](https://huggingface.co/papers/2605.10268)

- **Key result:** Superior performance with maintained data efficiency; no exact accuracy numbers in abstract.

- **Problem:** VLMs degrade on long documents; continued pre-training without data mixing loses short-context performance.
- **Method:** Strategic data mixture design for long-context continued pre-training of VLMs.
- **Key result:** Maintains performance across diverse contexts beyond 128K tokens; no exact numbers.

- **Problem:** Many-shot ICL scaling behavior differs from standard tasks; demonstration ordering is underexplored.

- **Key result:** Ordering and selection significantly impact performance; scaling not monotonic for reasoning tasks.
