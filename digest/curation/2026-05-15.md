## Part 2 — Research Radar (HuggingFace Papers, English)

### TL;DR — Today's Research Wind

Agent memory is this week's dominant thread: four independent papers (PREPING, EvolveMem, STALE, BOOKMARKS) attack the problem from orthogonal angles — cold-start, self-evolving retrieval, belief invalidation, and storyline compression. Simultaneously, the agent eval space is maturing fast with WildClawBench and FutureSim both pushing toward real-world, long-horizon measurement. Post-training via self-distillation (SDAIRL, Adaptive Teacher) and RL-based environment construction (Learning to Build) suggest the "train the agent by letting it construct its own curriculum" paradigm is converging.

### Agent frameworks & self-evolution

#### 🟢 [2605.15155] Self-Distilled Agentic Reinforcement Learning
- **Problem:** RL gives coarse trajectory-level rewards; no dense token-level guidance for long-horizon agents.
- **Method:** On-Policy Self-Distillation (OPSD) — teacher sees reference solution, distills dense supervision into RL rollouts.
- **Key result:** Outperforms vanilla RL baselines on long-horizon agentic benchmarks; specific gains not reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15155) · [HF](https://huggingface.co/papers/2605.15155)

#### 🟢 [2605.13880] PREPING: Building Agent Memory without Tasks
- **Problem:** Agents face cold-start gap when entering new environment with zero prior task experience.
- **Method:** Pre-deployment memory construction via environment exploration — no task signal required, builds memory structure offline.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13880) · [HF](https://huggingface.co/papers/2605.13880)

#### 🟢 [2605.13941] EvolveMem: Self-Evolving Memory Architecture via AutoResearch for LLM Agents
- **Problem:** Agent memory systems freeze retrieval logic at deployment; stored content grows but scoring/fusion/generation stay static.
- **Method:** AutoResearch loop — agent continuously proposes, tests, and adopts improvements to its own memory architecture.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13941) · [HF](https://huggingface.co/papers/2605.13941)

#### 🟢 [2605.14169] BOOKMARKS: Efficient Active Storyline Memory for Role-playing
- **Problem:** RPA memory via recurrent summarization loses critical details; profiling compresses too aggressively.
- **Method:** Search-based active memory — maintains storyline "bookmarks" that preserve key events without lossy compression.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14169) · [HF](https://huggingface.co/papers/2605.14169)

#### 🟢 [2605.15040] Orchard: An Open-Source Agentic Modeling Framework
- **Problem:** Open research on agentic LLMs bottlenecked by infra and training gaps vs. proprietary systems.
- **Method:** Open-source framework covering planning, reasoning, tool use, multi-turn interaction for agentic models.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15040) · [HF](https://huggingface.co/papers/2605.15040)

#### 🟢 [2605.14892] Beyond Individual Intelligence: Surveying Multi-Agent Collaboration, Failure Attribution, and Self-Evolution
- **Problem:** Single agents limited on tasks requiring sustained coordination across roles, tools, and environments.
- **Method:** Survey of collaboration architectures, failure attribution mechanisms, and self-evolution strategies in LLM MAS.
- **Key result:** No quantitative result reported (survey).
- *Links:* [PDF](https://arxiv.org/pdf/2605.14892) · [HF](https://huggingface.co/papers/2605.14892)

#### 🟡 [2605.14389] Nexus: An Agentic Framework for Time Series Forecasting
- **Problem:** Time series forecasting ignores unstructured contextual data (news, events) that humans naturally use.
- **Method:** Agentic pipeline that routes numerical + textual signals through specialized LLM and TSFM sub-agents.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14389) · [HF](https://huggingface.co/papers/2605.14389)

### Agent reliability, evaluation & safety

#### 🟢 [2605.10912] WildClawBench: A Benchmark for Real-World, Long-Horizon Agent Evaluation
- **Problem:** Existing agent benchmarks use synthetic sandboxes, short-horizon tasks, and mock APIs — not real CLI environments.
- **Method:** Real-world CLI-harness benchmark with long-horizon tasks and final + intermediate verification checks.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10912) · [HF](https://huggingface.co/papers/2605.10912)

#### 🟢 [2605.06527] STALE: Can LLM Agents Know When Their Memories Are No Longer Valid?
- **Problem:** Agent benchmarks test static fact retrieval; they ignore whether agents can revise stored beliefs when evidence changes.
- **Method:** STALE benchmark — tasks requiring agents to detect and update stale/invalidated memories over time.
- **Key result:** Current agents fail significantly at belief-update tasks; exact numbers not in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.06527) · [HF](https://huggingface.co/papers/2605.06527)

#### 🟢 [2605.15188] FutureSim: Replaying World Events to Evaluate Adaptive Agents
- **Problem:** No efficient way to measure whether agents adapt to new information in dynamic open-ended environments.
- **Method:** Grounded simulation replaying real-world events in chronological order to test agent adaptation.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15188) · [HF](https://huggingface.co/papers/2605.15188)

#### 🟡 [2605.14454] LiSA: Lifelong Safety Adaptation via Conservative Policy Induction
- **Problem:** Guardrails for deployed agents (reading private data, calling tools, multi-step workflows) must adapt without retraining.
- **Method:** Conservative Policy Induction — lifelong learning approach that updates safety policy from few examples without regression.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14454) · [HF](https://huggingface.co/papers/2605.14454)

### Post-training (RL, distillation, merging, calibration)

#### 🟡 [2605.13301] Achieving Gold-Medal-Level Olympiad Reasoning via Simple and Unified Scaling
- **Problem:** Existing reasoning models hit gold-medal performance on IMO/IPhO but with complex pipelines — what's the minimal recipe?
- **Method:** Simple, unified scaling of RL-based post-training without specialized tricks; systematic study of scaling axes.
- **Key result:** Gold-medal level on IMO and IPhO problems; specific score not stated in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13301) · [HF](https://huggingface.co/papers/2605.13301)

#### 🟢 [2605.14386] Darwin Family: MRI-Trust-Weighted Evolutionary Merging for Training-Free Scaling
- **Problem:** Can frontier-level reasoning be improved without any training, by reorganizing latent capabilities already in weights?
- **Method:** Gradient-free weight-space recombination guided by MRI (Model Reliability Index) trust weighting; evolutionary search.
- **Key result:** Improves reasoning on multiple benchmarks without additional training; exact delta not in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14386) · [HF](https://huggingface.co/papers/2605.14386)

#### 🟢 [2605.11458] Adaptive Teacher Exposure for Self-Distillation in LLM Reasoning
- **Problem:** On-policy self-distillation always exposes teacher to full reference solution — this design choice has never been questioned.
- **Method:** Adaptive Teacher Exposure — vary how much reference solution the teacher sees, controlling guidance intensity dynamically.
- **Key result:** Improves over standard OPSD; specific metric not reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.11458) · [HF](https://huggingface.co/papers/2605.11458)

#### 🟢 [2605.14392] Learning to Build the Environment: Self-Evolving Reasoning RL via Verifiable Environment Synthesis
- **Problem:** Self-improving LLMs generate problems/traces to imitate — can the model instead construct the environments that train it?
- **Method:** Environment-construction loop — model synthesizes verifiable RL environments, then trains on them; zero external data.
- **Key result:** No quantitative result reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14392) · [HF](https://huggingface.co/papers/2605.14392)

#### 🟢 [2605.08703] RewardHarness: Self-Evolving Agentic Post-Training
- **Problem:** Reward models for instruction-guided edits require large-scale preference annotations; humans can judge from few examples.
- **Method:** Agentic post-training pipeline where reward model self-evolves from few-shot human preference examples.
- **Key result:** Matches or exceeds fully-supervised reward models with far fewer annotations; exact numbers not in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.08703) · [HF](https://huggingface.co/papers/2605.08703)

### Inference optimization & compiler

#### 🟢 [2605.09681] Forcing-KV: Hybrid KV Cache Compression for Efficient Autoregressive Video Diffusion
- **Problem:** Autoregressive video diffusion models have O(n²) attention complexity from growing KV cache; streaming generation stalls.
- **Method:** Hybrid KV cache strategy — combines forced eviction and selective retention based on attention pattern analysis.
- **Key result:** Significant inference speedup on AR video generation; specific ratio not in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.09681) · [HF](https://huggingface.co/papers/2605.09681)

#### 🟢 [2605.14438] BEAM: Binary Expert Activation Masking for Dynamic Routing in MoE
- **Problem:** Standard Top-K MoE routing activates fixed experts per token — redundant compute, suboptimal inference latency.
- **Method:** Binary activation masks that learn to skip experts dynamically per token, beyond fixed Top-K.
- **Key result:** Reduces active compute without accuracy regression; exact FLOP reduction not in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14438) · [HF](https://huggingface.co/papers/2605.14438)

### Retrieval / RAG / memory

#### 🟢 [2605.14906] MemLens: Benchmarking Multimodal Long-Term Memory in Large Vision-Language Models
- **Problem:** No existing benchmark systematically compares long-context LVLMs vs. memory-augmented agents on multimodal tasks.
- **Method:** MemLens benchmark — tasks that require genuine multimodal memory, not answerable from text traces alone.
- **Key result:** Memory-augmented agents and long-context LVLMs have complementary failure modes; specific accuracy delta not reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14906) · [HF](https://huggingface.co/papers/2605.14906)

#### 🟢 [2605.15128] MemEye: A Visual-Centric Evaluation for Multimodal Agent Memory
- **Problem:** Existing memory evals let agents bypass visual evidence — questions answerable from captions alone.
- **Method:** MemEye — forces visual grounding; tests whether agents preserve and retrieve visual evidence for later reasoning.
- **Key result:** State-of-the-art agents perform significantly worse when visual shortcuts are removed; exact numbers not in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15128) · [HF](https://huggingface.co/papers/2605.15128)

#### 🟡 [2605.00180] RouteProfile: Design Space of LLM Profiles for Routing
- **Problem:** LLM routing research focuses on router mechanism, not on what capability profiles to build for each model.
- **Method:** Systematic study of how to construct and represent LLM profiles (capability fingerprints) for accurate routing.
- **Key result:** Profile design choices matter as much as router architecture; specific routing accuracy not stated.
- *Links:* [PDF](https://arxiv.org/pdf/2605.00180) · [HF](https://huggingface.co/papers/2605.00180)
