## Part 2 — Research Radar (HuggingFace Papers, English)

> ⚠️ HF date-specific endpoint returned 0 for 2026-05-17 (Sunday — HF doesn't curate weekend daily lists). Used the rolling-latest endpoint; 50 papers scanned, most submitted 2026-05-12 → 2026-05-13. Filtered to 13 keepers per `screening-criteria.md` v1.

### TL;DR — Today's Research Wind
Three convergent themes dominate this batch. (1) **Self-distillation is back, hard** — OPSD for multi-turn agents, Adaptive Teacher Exposure, and continual-learning self-distillation all argue that on-policy self-distillation is the new default supervision regime for post-training, replacing pure trajectory-level RL. (2) **Agent memory benchmarking is fragmenting on purpose** — STALE (knowing when memory is stale), MemLens / MemEye (multimodal grounding), and PREPING (cold-start pre-task memory) each carve out a distinct failure mode rather than competing on the same leaderboard, signaling that "agent memory" as a category is finally getting decomposed into engineering-tractable subproblems. (3) **Evolutionary / training-free model merging** crossed a meaningful capability bar — Darwin Family claims gradient-free weight-space recombination that approaches frontier reasoning without further training, which if real should change how small labs and indie devs think about closing the gap to frontier.

### Agent Reliability, Evaluation & Memory

#### 🟢 [2605.06527] STALE: Can LLM Agents Know When Their Memories Are No Longer Valid?
- **Problem:** Agents fail at implicit conflict — later observations invalidate earlier memory without explicit negation.
- **Method:** STALE benchmark + CUPMem method for premise-resistant memory revision via implicit policy adaptation.
- **Key result:** No quantitative number cited in abstract; identifies implicit-conflict as systematic blind spot across current memory systems.
- *Links:* [PDF](https://arxiv.org/pdf/2605.06527) · [HF](https://huggingface.co/papers/2605.06527)

#### 🟢 [2605.14906] MemLens: Benchmarking Multimodal Long-Term Memory in LVLMs
- **Problem:** No benchmark systematically compares long-context LVLMs vs. memory-augmented agents on truly multimodal queries.
- **Method:** MEMLENS benchmark over multi-session conversations requiring cross-modal token-counting and multi-session visual reasoning.
- **Key result:** No quantitative summary in abstract excerpt; positions memory-augmented agents vs. long-context as currently empirically open.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14906) · [HF](https://huggingface.co/papers/2605.14906)

#### 🟢 [2605.15128] MemEye: Visual-Centric Evaluation for Multimodal Agent Memory
- **Problem:** Existing evals let agents answer "visual" questions from captions/text traces alone, masking visual-evidence loss.
- **Method:** Evaluation framework that probes whether agents preserve fine-grained visual evidence; tests routing, temporal tracking, detail extraction.
- **Key result:** No quantitative number reported in abstract.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15128) · [HF](https://huggingface.co/papers/2605.15128)

#### 🟢 [2605.13880] PREPING: Building Agent Memory without Tasks
- **Problem:** Agents face a cold-start gap when entering new environments with no prior interaction or curated demos.
- **Method:** Pre-task memory construction via proposer-guided synthetic practice with solver and validator components.
- **Key result:** No quantitative result reported in abstract excerpt.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13880) · [HF](https://huggingface.co/papers/2605.13880)

#### 🟢 [2605.13941] EvolveMem: Self-Evolving Memory Architecture for LLM Agents
- **Problem:** Memory systems freeze scoring, fusion, and answer policies at deployment while only stored content evolves.
- **Method:** Co-evolution at two levels (knowledge + retrieval mechanism) via structured action space, LLM diagnosis module, guarded meta-analyzer.
- **Key result:** No quantitative result reported in abstract excerpt.
- *Links:* [PDF](https://arxiv.org/pdf/2605.13941) · [HF](https://huggingface.co/papers/2605.13941)

#### 🟡 [2605.10912] WildClawBench: Real-World, Long-Horizon Agent Evaluation
- **Problem:** Agent benchmarks rely on synthetic sandboxes, mock APIs, short horizons, final-answer checks — diverging from production CLI runtimes.
- **Method:** Native-runtime evaluation in Docker, with LLM/VLM judge providing semantic verification of intermediate tool calls.
- **Key result:** No quantitative summary cited in abstract excerpt; establishes evaluation methodology.
- *Links:* [PDF](https://arxiv.org/pdf/2605.10912) · [HF](https://huggingface.co/papers/2605.10912)

### Post-training (Self-Distillation, RL, Model Merging)

#### 🟢 [2605.15155] Self-Distilled Agentic Reinforcement Learning
- **Problem:** Trajectory-level RL reward gives only coarse supervision for long-horizon multi-turn agents.
- **Method:** On-Policy Self-Distillation (OPSD) with token-level guidance from a privileged-context teacher branch, skill-conditioned for agents.
- **Key result:** No quantitative number in abstract excerpt; presents OPSD as a denser auxiliary objective alongside RL.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15155) · [HF](https://huggingface.co/papers/2605.15155)

#### 🟢 [2605.14386] Darwin Family: Trust-Weighted Evolutionary Merging for Training-Free Reasoning
- **Problem:** Improving frontier-level reasoning normally requires more training/data; checkpoints encode latent capability not reorganized.
- **Method:** Training-free evolutionary merging via gradient-free weight-space recombination, 14-dim merge genome, MRI-Trust Fusion, Architecture Mapper for cross-architecture breeding.
- **Key result:** No quantitative number in abstract excerpt; claim is training-free reasoning gains.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14386) · [HF](https://huggingface.co/papers/2605.14386)

#### 🟡 [2605.11458] Adaptive Teacher Exposure for Self-Distillation in LLM Reasoning
- **Problem:** Self-distillation default — teacher always sees full reference reasoning — induces exposure mismatch in student rollouts.
- **Method:** Beta-policy controller scheduling teacher visibility with discounted learning-progress reward for delayed credit assignment.
- **Key result:** No quantitative number in abstract excerpt.
- *Links:* [PDF](https://arxiv.org/pdf/2605.11458) · [HF](https://huggingface.co/papers/2605.11458)
