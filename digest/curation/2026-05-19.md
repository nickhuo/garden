## Part 2 ‚ÄĒ Research Radar (HuggingFace Papers, English)

### TL;DR ‚ÄĒ Today's Research Wind

Inference-side efficiency dominated the queue: three different angles on the same problem ‚ÄĒ **NVFP4 long-video infra** (LongLive-2.0), **chunked-prefill block-union KV selection** (CompactAttention), and **post-merge quantization with expert anchoring** (E-PMQ) ‚ÄĒ collectively say "quantization plus structural sparsity is now the default optimization stack, not an afterthought." Meanwhile the agent-reliability side delivered three honest negative-result papers: tool-use *knowing-doing gap* (2605.14038), economic alignment failure modes in multi-agent marketplaces (Agent Bazaar), and a 32% Opus-4.6 score on the new omni-modal tool-use benchmark TOBench. The synthesis worth weighing in on publicly: **agent harnesses are now the real product** ‚ÄĒ both *Code as Agent Harness* (survey) and *SkillsVote* (lifecycle governance) frame harness, not model, as the unit of capability ‚ÄĒ directly aligned with Nick's Donut Labs work.

### Agent frameworks & self-evolution

### ūüüĘ [2605.18401] SkillsVote ‚ÄĒ Lifecycle Governance of Agent Skills
- **Problem:** Open Agent-Skill ecosystems are redundant, noisy, and pollute future context with bad updates.
- **Method:** Lifecycle-governance pipeline: agentic library search ‚Üí trajectory decomposition ‚Üí evidence-gated skill admission.
- **Key result:** +7.9pp on Terminal-Bench 2.0 (GPT-5.2 offline evolution); +2.6pp on SWE-Bench Pro online.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18401) ¬∑ [HF](https://huggingface.co/papers/2605.18401) ¬∑ [GitHub](https://github.com/MemTensor/skills-vote) ‚≠ź201

### ūüüĘ [2605.18747] Code as Agent Harness (survey)
- **Problem:** Agent infra is fragmented ‚ÄĒ no unified framing for code's role as harness.
- **Method:** Three-layer taxonomy: harness interface (reason/act/env) + mechanisms (plan/mem/tools) + multi-agent scaling.
- **Key result:** No quantitative result reported (survey).
- *Links:* [PDF](https://arxiv.org/pdf/2605.18747) ¬∑ [HF](https://huggingface.co/papers/2605.18747) ¬∑ [GitHub](https://github.com/YennNing/Awesome-Code-as-Agent-Harness-Papers) ‚≠ź17

### ūüüĘ [2605.17933] AtlasVA ‚ÄĒ Self-Evolving Visual Skill Memory for Teacher-Free VLM Agents
- **Problem:** Text-only agent memory is lossy for spatial decisions; most frameworks depend on a proprietary teacher.
- **Method:** Three-layer visual memory (heatmaps + exemplars + symbolic skills) + potential-based shaping rewards from self-evolving danger/affinity atlases.
- **Key result:** Outperforms text-centric memory on Sokoban / FrozenLake / 3D nav / 3D manipulation. No single headline metric reported.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17933) ¬∑ [HF](https://huggingface.co/papers/2605.17933)

### Agent reliability, evaluation & safety

### ūüüĘ [2605.14038] Model-Adaptive Tool Necessity ‚ÄĒ The Knowing-Doing Gap
- **Problem:** Tool-necessity datasets are model-agnostic, but capability boundaries are model-specific.
- **Method:** Empirically-grounded model-adaptive necessity definition + hidden-state probes separating cognition vs. execution stages.
- **Key result:** 26.5‚Äď54.0% mismatch (arithmetic), 30.8‚Äď41.8% (factual QA); failure concentrated in cognition‚Üíaction transition, not cognition itself.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14038) ¬∑ [HF](https://huggingface.co/papers/2605.14038) ¬∑ [GitHub](https://github.com/chengez/Tool-Cognition-Action) ‚≠ź4

### ūüüĘ [2605.17698] Agent Bazaar ‚ÄĒ Economic Alignment in Multi-Agent Marketplaces
- **Problem:** LLM-as-economic-agent introduces collective failure modes (volatility, sybil deception).
- **Method:** Two scenarios (B2C "Crash", C2C "Lemon Market") + Economic Alignment Score (EAS, 4-component) + REINFORCE++ adaptive curriculum.
- **Key result:** Trained 9B beats all evaluated frontier and open-weight models on EAS; failure severity varies by model, not by size.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17698) ¬∑ [HF](https://huggingface.co/papers/2605.17698)

### ūüüĘ [2605.16909] TOBench (MM-ToolBench) ‚ÄĒ Task-Oriented Omni-Modal Tool-Using Benchmark
- **Problem:** Existing tool-use benchmarks evaluate tools/vision/reasoning in isolation, not closed-loop omni-modal.
- **Method:** 100 executable tasks across 20 subcategories on 27 MCP servers / 324 tools; closed-loop multimodal verification with grounded evaluators.
- **Key result:** Claude Opus 4.6 = 32.0% task success vs. 94.0% human ‚ÄĒ large headroom on real omni-modal tool use.
- *Links:* [PDF](https://arxiv.org/pdf/2605.16909) ¬∑ [HF](https://huggingface.co/papers/2605.16909) ¬∑ [GitHub](https://github.com/Pi3AI/TOBench)

### ūüü° [2605.18549] Monitoring the Internal Monologue ‚ÄĒ Probe Trajectories Reveal Reasoning Dynamics
- **Problem:** CoT is not always faithful ‚Üí unreliable as a safety-monitoring substrate.
- **Method:** Per-token probes form trajectories; signal-processing features (volatility/trend/steady-state) + max-pooling.
- **Key result:** Up to 95% AUROC predicting future model behavior; average/last-token pooling near-random.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18549) ¬∑ [HF](https://huggingface.co/papers/2605.18549)

### ūüüĘ [2605.16819] AgentKernelArena ‚ÄĒ Generalization-Aware Benchmarking of GPU Kernel Optimization Agents
- **Problem:** Existing kernel benchmarks measure single LLM calls, not full agent workflows; no unseen-config generalization test.
- **Method:** 196 tasks across HIP‚ÜĒHIP / Triton‚ÜĒTriton / PyTorch‚ÜíHIP; gated compile + correctness + perf + unseen-config protocol.
- **Key result:** Up to 6.89√ó speedup PyTorch‚ÜíHIP; HIP‚ÜĒHIP and Triton‚ÜĒTriton transfer to unseen shapes; PyTorch‚ÜíHIP fails generalization (agents hardcode shape).
- *Links:* [PDF](https://arxiv.org/pdf/2605.16819) ¬∑ [HF](https://huggingface.co/papers/2605.16819) ¬∑ [GitHub](https://github.com/AMD-AGI/AgentKernelArena) ‚≠ź16

### Post-training (RL, distillation, merging, calibration)

### ūüüĘ [2605.18643] ZEDA ‚ÄĒ Post-Trained MoE Can Skip Half Experts via Self-Distillation
- **Problem:** Dynamic MoE methods need from-scratch pretraining; converting fully-trained static MoE is underexplored.
- **Method:** Inject parameter-free zero-output experts + two-stage self-distillation with group-level balancing loss.
- **Key result:** >50% expert FLOPs eliminated at marginal accuracy loss on Qwen3-30B-A3B / GLM-4.7-Flash; +6.1 / +4.0 over best dynamic MoE baseline; ~1.20√ó inference speedup.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18643) ¬∑ [HF](https://huggingface.co/papers/2605.18643) ¬∑ [GitHub](https://github.com/TsinghuaC3I/ZEDA) ‚≠ź5

### ūüü° [2605.16865] MixSD ‚ÄĒ Mixed Contextual Self-Distillation for Knowledge Injection
- **Problem:** SFT degrades pretrained capability because external targets diverge from the model's autoregressive distribution.
- **Method:** Mix tokens from expert (fact-conditioned) and naive conditionals of the *same* base model to construct supervision dynamically.
- **Key result:** Retains up to 100% of held-out capability vs. ‚Č§1% for standard SFT; lower-NLL targets, less Fisher-direction drift.
- *Links:* [PDF](https://arxiv.org/pdf/2605.16865) ¬∑ [HF](https://huggingface.co/papers/2605.16865)

### ūüüĘ [2605.16882] E-PMQ ‚ÄĒ Expert-Guided Post-Merge Quantization
- **Problem:** Applying PTQ to a merged-expert model couples quantization deviation with merging deviation ‚Üí unreliable.
- **Method:** Expert-guided layer-wise calibration + merged-weight anchoring.
- **Key result:** 4-bit GPTQ on CLIP-ViT-B/32 8-task: 65.0‚Üí73.6% (Task Arithmetic), 69.1‚Üí74.8% (TIES); 20-task CLIP-ViT-L/14: 34.8‚Üí76.7%.
- *Links:* [PDF](https://arxiv.org/pdf/2605.16882) ¬∑ [HF](https://huggingface.co/papers/2605.16882) ¬∑ [GitHub](https://github.com/wwjzhy/E-PMQ) ‚≠ź2

### Inference optimization & compiler

### ūüüĘ [2605.18739] LongLive-2.0 ‚ÄĒ NVFP4 Parallel Infrastructure for Long Video Generation
- **Problem:** Long-video training/inference hits speed and memory ceilings on existing precision/parallelism stacks.
- **Method:** Balanced sequence-parallel AR training + NVFP4 (W4A4) + NVFP4 KV-cache + async streaming VAE decode on Blackwell.
- **Key result:** 2.15√ó training speedup, 1.84√ó inference speedup; LongLive-2.0-5B at 45.7 FPS ‚ÄĒ first NVFP4 system for long video.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18739) ¬∑ [HF](https://huggingface.co/papers/2605.18739) ¬∑ [GitHub](https://github.com/NVlabs/LongLive) ‚≠ź1,221

### ūüüĘ [2605.16839] CompactAttention ‚ÄĒ Block-Union KV Selection for Chunked Prefill
- **Problem:** Existing sparse-attention is designed for one-shot prefill; doesn't carry to chunked prefill efficiently.
- **Method:** Treat 2D block-sparse masks as KV-selection signals (not exec plans); convert to GQA-aware per-group KV block tables via Q-block + intra-group union.
- **Key result:** Up to 2.72√ó attention speedup at 128K ctx on LLaMA-3.1-8B-Instruct, accuracy close to dense on RULER.
- *Links:* [PDF](https://arxiv.org/pdf/2605.16839) ¬∑ [HF](https://huggingface.co/papers/2605.16839) ¬∑ [GitHub](https://github.com/jiwonsong-dev/CompactAttention)

### ūüü° [2605.17672] PUMA ‚ÄĒ Semantic-Preserving Early Exit for Reasoning Models
- **Problem:** Answer-level early-exit signals fire before reasoning has actually converged ‚Üí degraded accuracy.
- **Method:** Lightweight Redundancy Detector flags semantically redundant continuation, paired with answer-level verification.
- **Key result:** 26.2% average token reduction at preserved accuracy across 5 LRMs √ó 5 reasoning benchmarks.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17672) ¬∑ [HF](https://huggingface.co/papers/2605.17672) ¬∑ [GitHub](https://github.com/giovanni-vaccarino/PUMA) ‚≠ź2

### ūüü° [2605.17260] LiteFrame ‚ÄĒ Efficient Vision Encoders for Frame Scaling in Video LLMs
- **Problem:** Post-hoc token reduction shifts latency from LLM to per-frame vision-encoder cost.
- **Method:** Compressed Token Distillation (CTD): compact student encoder directly predicts teacher's spatio-temporally compressed reps; + Language Model Adaptation.
- **Key result:** 35% end-to-end latency reduction vs. InternVL3-8B while processing 8√ó more frames; improved average video accuracy.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17260) ¬∑ [HF](https://huggingface.co/papers/2605.17260) ¬∑ [GitHub](https://github.com/jjihwan/LiteFrame)

### ūüü° [2605.14589] EndPrompt ‚ÄĒ Long-Context Extension via Terminal Anchoring
- **Problem:** Long-context fine-tuning costs quadratic memory because it needs full-length sequences.
- **Method:** Append a brief "terminal prompt" with high positional indices to short training sequences ‚Üí exposes long-range RoPE distances without long inputs.
- **Key result:** LLaMA 8K‚Üí64K: RULER 76.03 average; outperforms LCEG (72.24), LongLoRA (72.95), full-length FT (69.23) at lower compute.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14589) ¬∑ [HF](https://huggingface.co/papers/2605.14589)

### Code / web agents

### ūüüĘ [2605.17242] TDDev ‚ÄĒ Multi-Agent TDD for Full-Stack Web Apps
- **Problem:** Coding-agent web apps fail functional requirements in >70% of cases; correctness needs deploy+browser verification.
- **Method:** 3-stage closed loop: requirements‚Üíacceptance tests; deploy + browser-interaction simulation; failures‚Üístructured repair reports.
- **Key result:** +34‚Äď48pp over no-TDD baseline. Critical finding: protocol must match generation style; mismatched TDD eliminates benefit and 25√ó the token cost.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17242) ¬∑ [HF](https://huggingface.co/papers/2605.17242) ¬∑ [GitHub](https://github.com/yxwan123/TDDev) ‚≠ź10

### ūüü° [2605.17283] OProver ‚ÄĒ Unified Agentic Formal Theorem Proving in Lean 4
- **Problem:** Agentic proving usually only at inference time, not integrated into prover training.
- **Method:** Continued pretraining + iterative post-training: agentic proving ‚Üí SFT on repairs + RL on hard cases; OProofs corpus (1.77M statements / 6.86M verified proofs).
- **Key result:** OProver-32B Pass@32: MiniF2F 93.3%, ProverBench 58.2%, PutnamBench 11.3% ‚ÄĒ top open-weight whole-proof prover.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17283) ¬∑ [HF](https://huggingface.co/papers/2605.17283) ¬∑ [GitHub](https://github.com/multimodal-art-projection/OProver) ‚≠ź1
