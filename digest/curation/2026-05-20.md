## Part 2 — Research Radar (HuggingFace Papers, English)

### TL;DR — Today's Research Wind

Today's 33 papers split cleanly: a heavy **post-training / RL** block where the recurring theme is *fixing the credit-assignment problem in RLVR* — three separate papers (AntiSD, CEPO, CopT) all attack the same flaw: every token in a correct rollout gets the same reward regardless of whether it was a decisive reasoning step or filler. The second front is **agent environments & evaluation** — EnvFactory, OpenComputer, OmniGUI, AutoResearchClaw all converge on the same bottleneck: agents need *verifiable, executable environments* to train and be measured against, and frontier agents still fail end-to-end despite partial progress. Inference and long-context memory round it out (Graft, PEEK, AttentionMemory). 22 of 33 kept.

### Agent frameworks & self-evolution

### 🟢 [2605.20025] AutoResearchClaw — self-reinforcing autonomous research
- **Problem:** Autonomous research systems are linear pipelines: single-agent, stop on failure, no cross-run learning.
- **Method:** Multi-agent debate + self-healing executor with Pivot/Refine loop + verifiable reporting + 7-mode human-in-the-loop + cross-run evolution.
- **Key result:** Outperforms AI Scientist v2 by 54.7% on ARC-Bench (25-topic benchmark).
- *Links:* [PDF](https://arxiv.org/pdf/2605.20025) · [HF](https://huggingface.co/papers/2605.20025) · [GitHub](https://github.com/aiming-lab/AutoResearchClaw)

### 🟢 [2605.18703] EnvFactory — scaling tool-use agents via executable env synthesis
- **Problem:** Agentic RL is bottlenecked by lack of scalable executable environments and realistic training data.
- **Method:** Auto-explores/verifies stateful tool environments from real resources; synthesizes multi-turn trajectories via topology-aware sampling + calibrated refinement.
- **Key result:** With only 85 environments, +15% on BFCLv3, +8.6% on MCP-Atlas, +6% on τ²-Bench/VitaBench for Qwen3.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18703) · [HF](https://huggingface.co/papers/2605.18703) · [GitHub](https://github.com/LARK-AI-Lab/EnvFactory) ⭐16

### Agent reliability, evaluation & safety

### 🟢 [2605.19769] OpenComputer — verifiable software worlds for computer-use agents
- **Problem:** Computer-use agents lack verifiable environments; LLM-as-judge evaluation is unreliable for fine-grained state.
- **Method:** Verifier-grounded framework: app-specific state verifiers + self-evolving verification layer + task-generation pipeline + auditable partial-credit harness.
- **Key result:** Hard-coded verifiers align with humans better than LLM-as-judge; covers 33 apps / 1,000 tasks; frontier agents still fail end-to-end.
- *Links:* [PDF](https://arxiv.org/pdf/2605.19769) · [HF](https://huggingface.co/papers/2605.19769) · [GitHub](https://github.com/echo0715/OpenComputer) ⭐3

### 🟢 [2605.18758] OmniGUI — benchmarking GUI agents in omni-modal phone environments
- **Problem:** GUI-agent benchmarks rely on static screenshots, ignoring transient audio and temporal video cues.
- **Method:** Step-level benchmark with interleaved image+audio+video at every action step; 709 episodes (2,579 steps), 29 apps.
- **Key result:** Action-prediction degrades sharply under synchronous audio/temporal signals; cross-modal interference from irrelevant noise.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18758) · [HF](https://huggingface.co/papers/2605.18758) · [GitHub](https://github.com/omni-gui/OmniGUI) ⭐2

### 🟢 [2605.18827] CGR — code-guided reasoning for small language models
- **Problem:** SLMs are evaluated as direct answerers, but deployed systems use code/tool scaffolds.
- **Method:** Code-Guided Reasoning protocol: normalized item interface, generator prompt, Python scaffold, solver-call helpers, three-channel result record.
- **Key result:** 66.21% macro assisted accuracy vs 38.11% direct (+28.10 pts; +14.11 under stricter direct-signal gate).
- *Links:* [PDF](https://arxiv.org/pdf/2605.18827) · [HF](https://huggingface.co/papers/2605.18827)

### 🟡 [2605.17360] Omni-DuplexEval — evaluating real-time duplex omni-modal interaction
- **Problem:** MLLMs are evaluated offline; no benchmark for real-time duplex (streaming-in, respond-when-appropriate) interaction.
- **Method:** Benchmark of 660 videos with temporal metadata; LLM-as-Judge framework scoring both content alignment and response timing.
- **Key result:** Best model only 39.6% overall, 20.0% on Proactive Reminder — models fail at *when* and *what* to respond.
- *Links:* [PDF](https://arxiv.org/pdf/2605.17360) · [HF](https://huggingface.co/papers/2605.17360) · [GitHub](https://github.com/OpenBMB/Omni-DuplexEval) ⭐2

### 🟡 [2605.18646] Language-switching backdoor circuit
- **Problem:** Internal mechanism of how a trigger sequence hijacks LLM computation is poorly understood.
- **Method:** Mechanistic circuit analysis of a 9-token Latin trigger redirecting English→French output in an 8B autoregressive model.
- **Key result:** Trigger propagates in a subspace *orthogonal* to the natural language-identity direction — defenses scanning for language-like signals would miss it entirely.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18646) · [HF](https://huggingface.co/papers/2605.18646)

### Post-training (RL, distillation, merging, calibration)

### 🟢 [2605.19577] GoLongRL — capability-oriented long-context RL
- **Problem:** Long-context RL methods over-focus on complex retrieval paths, yielding homogeneous tasks and weak rewards.
- **Method:** Open 23K-sample RLVR dataset spanning 9 task types + TMN-Reweight (task-level mean normalization + difficulty-adaptive weighting).
- **Key result:** Dataset alone beats closed-source QwenLong-L1.5; Qwen3-30B-A3B reaches DeepSeek-R1-0528-comparable long-context performance.
- *Links:* [PDF](https://arxiv.org/pdf/2605.19577) · [HF](https://huggingface.co/papers/2605.19577) · [GitHub](https://github.com/xiaoxuanNLP/GoLongRL) ⭐7

### 🟢 [2605.11609] AntiSD — anti-self-distillation for reasoning RL
- **Problem:** On-policy self-distillation gives inconsistent gains in math reasoning; privileged context inflates confidence on trivial tokens.
- **Method:** Anti-Self-Distillation *ascends* a divergence between student and teacher (reverses per-token sign); entropy-triggered gate disables it when teacher entropy collapses.
- **Key result:** Reaches GRPO baseline accuracy in 2–10× fewer steps; final accuracy up to +11.5 points across 4B–30B models.
- *Links:* [PDF](https://arxiv.org/pdf/2605.11609) · [HF](https://huggingface.co/papers/2605.11609) · [GitHub](https://github.com/FloyedShen/AntiSD) ⭐3

### 🟢 [2605.19436] CEPO — RLVR self-distillation via contrastive evidence
- **Problem:** In RLVR every token in a correct rollout gets the same reward regardless of being decisive or filler.
- **Method:** Contrastive Evidence Policy Optimization asks "does the correct answer favor this token *while the wrong answer disfavors it*?"; wrong-answer teacher built free from rejected rollouts.
- **Key result:** 43.43% / 60.56% avg accuracy at 2B/4B on 5 multimodal math benchmarks vs 41.17% / 57.43% for GRPO.
- *Links:* [PDF](https://arxiv.org/pdf/2605.19436) · [HF](https://huggingface.co/papers/2605.19436) · [GitHub](https://github.com/ahmedheakl/CEPO) ⭐2

### 🟢 [2605.15529] BetaPRM — process rewards with learned reliability
- **Problem:** PRMs output a single reward per step with no indication of when to trust it.
- **Method:** Distributional PRM learns a Beta belief via Beta-Binomial likelihood; Adaptive Computation Allocation uses the reliability signal in Best-of-N.
- **Key result:** Improves PRM-guided Best-of-N selection; ACA cuts token usage up to 33.57% while improving final accuracy.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15529) · [HF](https://huggingface.co/papers/2605.15529) · [GitHub](https://github.com/JinYuanLi0012/Beta-Binomial-PRM) ⭐2

### 🟢 [2605.20075] CopT — contrastive on-policy thinking
- **Problem:** Standard CoT forces thinking before answering, delaying plausible answers and burning tokens (performative reasoning).
- **Method:** Reverses order — draft answer first, then on-policy thinking conditioned on the draft; continuous embeddings act as inference-time contrastive verifiers.
- **Key result:** Up to +23% peak accuracy and −57% token usage across math/coding/agentic tasks, training-free.
- *Links:* [PDF](https://arxiv.org/pdf/2605.20075) · [HF](https://huggingface.co/papers/2605.20075) · [GitHub](https://github.com/sdc17/CopT) ⭐2

### 🟡 [2605.09640] RaPO — overcoming catastrophic forgetting with RFT
- **Problem:** RFT is more forgetting-resilient than SFT but still forgets in class/domain-incremental visual continual learning.
- **Method:** Retention-aware Policy Optimization: Retention Reward converts trajectory drift into a reward; Cross-Task Advantage Normalization stabilizes across task boundaries.
- **Key result:** Substantially reduces catastrophic forgetting while preserving plasticity across 5 visual continual-learning settings (no single headline metric).
- *Links:* [PDF](https://arxiv.org/pdf/2605.09640) · [HF](https://huggingface.co/papers/2605.09640)

### 🟡 [2605.18855] Delta Attention Residuals
- **Problem:** Attention Residuals attend over cumulative hidden states that are highly redundant, causing routing collapse in deep layers.
- **Method:** Route over *deltas* (per-sublayer change v_i = h_{i+1} − h_i) instead of cumulative states, yielding higher-contrast attention.
- **Key result:** 1.7–8.2% validation perplexity gains across 220M–7.6B; convertible from pretrained checkpoints via fine-tuning.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18855) · [HF](https://huggingface.co/papers/2605.18855)

### Inference optimization & compiler

### 🟢 [2605.20104] Graft — hybrid tree construction for speculative decoding
- **Problem:** Large draft trees in speculative decoding incur severe VRAM-bandwidth/compute overhead; dynamic pruning discards valid candidates.
- **Method:** Couples pruning and retrieval as mutually reinforcing — sequential "prune-then-graft" attaches retrieved tokens into pruning-opened positions; training-free, lossless.
- **Key result:** Up to 5.41× speedup on short-context; +21.8% avg speedup over EAGLE-3 on Qwen3-235B.
- *Links:* [PDF](https://arxiv.org/pdf/2605.20104) · [HF](https://huggingface.co/papers/2605.20104)

### Retrieval / RAG / memory

### 🟢 [2605.19932] PEEK — context map as orientation cache for long-context agents
- **Problem:** Existing approaches preserve trajectories or strategies but not reusable *orientation knowledge* about a recurring context.
- **Method:** Caches a constant-sized "context map" maintained by a programmable policy: Distiller, Cartographer, priority-based Evictor under a fixed token budget.
- **Key result:** +6.3–34.0% over baselines with 93–145 fewer iterations and 1.7–5.8× lower cost than ACE; generalizes to OpenAI Codex.
- *Links:* [PDF](https://arxiv.org/pdf/2605.19932) · [HF](https://huggingface.co/papers/2605.19932) · [GitHub](https://github.com/zhuohangu/peek)

### 🟢 [2605.18226] AttentionMemory — context memorization for efficient long-context generation
- **Problem:** Prefix influence fades during generation, and attention over the prefix scales linearly with length.
- **Method:** Attention-state memory — training-free externalization of the prefix into a lookup-based memory of precomputed attention states.
- **Key result:** Beats in-context learning at 1K–8K budgets; −1.36× attention latency at 8K; surpasses full-attention RAG using 20% of its memory.
- *Links:* [PDF](https://arxiv.org/pdf/2605.18226) · [HF](https://huggingface.co/papers/2605.18226) · [GitHub](https://github.com/yasu0001/AttentionMemory)

### 🟢 [2605.14236] IReranker — active learners as efficient PRP rerankers
- **Problem:** Pairwise Ranking Prompting judgments are noisy, order-sensitive, intransitive; sorting truncation gives unreliable top-K.
- **Method:** Reframe PRP reranking as active learning from noisy pairwise comparisons; randomized-direction oracle converts position bias into zero-mean noise.
- **Key result:** Improves NDCG@10 per call in the call-constrained regime; unbiased aggregate ranking with one LLM call per pair.
- *Links:* [PDF](https://arxiv.org/pdf/2605.14236) · [HF](https://huggingface.co/papers/2605.14236) · [GitHub](https://github.com/jerecoder/IReranker)

### Multimodal, world models, generation

### 🟡 [2605.16403] When Vision Speaks for Sound
- **Problem:** Video MLLMs fake audio understanding — they exploit visual-acoustic correlations instead of verifying the audio stream (audio-visual Clever Hans).
- **Method:** Thud, an intervention probing framework (Shift/Mute/Swap counterfactual audio edits) + two-stage alignment recipe with intervention-derived preference pairs.
- **Key result:** Best 10K-sample recipe improves average across three intervention dimensions by 28 percentage points.
- *Links:* [PDF](https://arxiv.org/pdf/2605.16403) · [HF](https://huggingface.co/papers/2605.16403) · [GitHub](https://github.com/rakanWen/wvs-code) ⭐1

### 🟡 [2605.18748] Aurora — unified video editing with a tool-using agent
- **Problem:** Unified video-editing models assume the user supplies model-ready text, references, and spatial grounding, which real requests omit.
- **Method:** Tool-augmented VLM agent maps raw requests to a structured edit plan; trained with supervised planning data + preference pairs for tool use.
- **Key result:** Improves over instruction-only baselines on AgentEdit-Bench; the VLM agent transfers to frozen video-editing models (no single metric).
- *Links:* [PDF](https://arxiv.org/pdf/2605.18748) · [HF](https://huggingface.co/papers/2605.18748) · [GitHub](https://github.com/yeates/Aurora) ⭐3

### 🟡 [2605.15458] VideoRLVR — video models can reason with verifiable rewards
- **Problem:** Video diffusion models are optimized for plausible generation, not verifiable reasoning under explicit constraints.
- **Method:** SDE-GRPO backbone + dense decomposed rewards + Early-Step Focus (RL only on early denoising phase).
- **Key result:** Early-Step Focus cuts training latency ~40%; beats SFT baselines on Maze/FlowFree/Sokoban and out-of-domain.
- *Links:* [PDF](https://arxiv.org/pdf/2605.15458) · [HF](https://huggingface.co/papers/2605.15458) · [GitHub](https://github.com/luka-group/VideoRLVR) ⭐1

### 🟡 [2605.18714] SGT — semantic generative tuning for unified multimodal models
- **Problem:** UMMs optimize understanding (sparse text signal) and generation (dense pixel objective) separately, yielding misaligned representations.
- **Method:** Semantic Generative Tuning uses image segmentation as a generative proxy to align understanding and generation.
- **Key result:** Consistently improves both multimodal comprehension and generative fidelity; improves feature linear separability (no single headline metric).
- *Links:* [PDF](https://arxiv.org/pdf/2605.18714) · [HF](https://huggingface.co/papers/2605.18714) · [GitHub](https://github.com/song2yu/SGT) ⭐28
