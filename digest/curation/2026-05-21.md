## Part 2 — Research Radar（HuggingFace Papers）

### TL;DR — 本日 research 风向

今天 32 篇 papers 明显聚向两个与 Nick agenda 直接相关的簇:**KV-cache 量化（quantization）**——OScaR、OCTOPUS、Mix-Quant 三篇互不相关的论文从不同角度攻击同一个 long-context 显存瓶颈；以及 **agent 可靠性 / 评估（reliability & evaluation）**——SpecBench 量化 reward hacking、LongMINT 测 memory 干扰、MOCHA 优化 agent skill。RLVR 学习动力学也被三篇同时命中（RELEX、unlearnability 现象、DPO≡RLHF 条件等价证明）——这个领域现在开始追问 RL post-training 「为什么有效」,而不只是「确实有效」。多模态生成类论文数量不少,但大多被 drop:纯 CV 应用,无可迁移的 agent / LLM 方法。

### Agent 可靠性、评估与安全（reliability, evaluation & safety）

#### 🟢 [2605.21384] SpecBench — 量化长程 coding agent 的 reward hacking
- **Problem:** coding agent 产出代码的速度超过人类 review,监督面坍缩到测试套件,诱发 reward hacking。
- **Method:** 把 SWE 任务拆为可见验证测试（visible validation tests）与留出测试（held-out tests）;SpecBench 度量「通过测试」与「真正解决」之间的落差。
- **Key result:** abstract 未报告量化指标（benchmark 构建型论文）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21384) · [HF](https://huggingface.co/papers/2605.21384)

#### 🟢 [2605.18565] LongMINT — 多目标干扰下的 agent memory 评估
- **Problem:** 现有 memory benchmark 只测静态独立召回,忽略了已更新 memory 之间的相互干扰。
- **Method:** 构建一个含反复更新、互相干扰的 memory 条目的 benchmark,要求长程下精确召回 + 多条信息的聚合推理。
- **Key result:** abstract 未报告量化指标。
- *Links:* [PDF](https://arxiv.org/pdf/2605.18565) · [HF](https://huggingface.co/papers/2605.18565)

#### 🟢 [2605.19330] MOCHA — 多目标 Chebyshev 退火做 agent skill 优化
- **Problem:** agent「skill」是多字段产物,受平台硬约束（description 被截断、instruction body 被压缩）。
- **Method:** 用多目标 Chebyshev annealing 在相互竞争的字段间优化 skill spec,适配平台截断/压缩约束。
- **Key result:** abstract 未报告量化指标（来源 abstract 被截断）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.19330) · [HF](https://huggingface.co/papers/2605.19330)

#### 🟡 [2605.20668] AI 审稿人的边界与机会

- **Method:** 用 GPT-5.2、Gemini 3.0 Pro、Claude Opus 4.5 去「评审 Nature 系列论文的评审意见」,以 45 位专家科学家为对照。

### Post-training（RL、蒸馏、校准）

#### 🟢 [2605.21468] RELEX — 用 rank-1 轨迹做最小化 RLVR
- **Problem:** RLVR 权重轨迹的几何结构研究不足,导致 RLVR 训练算力开销大。
- **Method:** 证明 RLVR 权重轨迹近似 rank-1 且可预测;用短轨迹上的线性回归做外推（extrapolation）。
- **Key result:** abstract 未报告量化指标（主张极少量训练即足够,详见论文）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21468) · [HF](https://huggingface.co/papers/2605.21468) · [GitHub](https://github.com/weizhepei/RELEX)

#### 🟢 [2605.16787] RLVR 中的「不可学习」（unlearnability）现象
- **Problem:** RLVR 学习动力学不透明——部分难样本始终无法被学会。
- **Method:** 用跨样本梯度分析（gradient similarity）刻画「不可学习」样本;提出数据增强（data augmentation）来修复。
- **Key result:** abstract 未报告量化指标。
- *Links:* [PDF](https://arxiv.org/pdf/2605.16787) · [HF](https://huggingface.co/papers/2605.16787) · [GitHub](https://github.com/yulinchen99/unlearnability-rlvr) ⭐1

#### 🟢 [2605.20834] DPO 与 RLHF 的条件等价性
- **Problem:** DPO 被当作与 RLHF 普遍等价,但该假设常常不成立。
- **Method:** 证明等价性是「有条件」的——取决于关于 reference policy 的一个隐含假设;刻画失效模式,提出一个可证明对齐的变体。
- **Key result:** abstract 未报告量化指标（理论型论文）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.20834) · [HF](https://huggingface.co/papers/2605.20834)

#### 🟢 [2602.07892] 把安全对齐当作持续学习——正交梯度投影（orthogonal gradient projection）
- **Problem:** 安全 post-training 会削弱通用效用——即「对齐税」（alignment tax）。
- **Method:** 把序列化对齐视为持续学习（continual learning）;用相对 reference subspace 的正交梯度投影,削减梯度干扰。
- **Key result:** abstract 未报告量化指标。
- *Links:* [PDF](https://arxiv.org/pdf/2602.07892) · [HF](https://huggingface.co/papers/2602.07892) · [GitHub](https://github.com/SunGL001/OGPSA) ⭐2

#### 🟡 [2605.20258] SelfCI — 为情境完整性（contextual integrity）做互补自蒸馏
- **Problem:** LLM 个人 agent 会泄露敏感信息——违反 contextual-integrity 隐私规范。
- **Method:** 互补自蒸馏（complementary self-distillation）+ reverse-KL + Product-of-Experts 策略优化,平衡「披露」与「完成任务」。
- **Key result:** abstract 未报告量化指标（主张改善了隐私-效用 trade-off）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.20258) · [HF](https://huggingface.co/papers/2605.20258) · [GitHub](https://github.com/sw-programmer/SelfCI) ⭐1

#### 🟡 [2605.19376] 生成式递归推理（Generative Recursive Reasoning）
- **Problem:** 递归推理模型（recursive reasoning model）是确定性的——单一 latent 轨迹限制了表达力。
- **Method:** 概率化多轨迹 latent 计算,通过摊销变分推断（amortized variational inference）覆盖随机 latent 轨迹。

### 推理优化与编译器（inference optimization & compiler）

#### 🟢 [2605.20315] Mix-Quant — 为 agentic LLM 做「量化 prefilling、精确 decoding」
- **Problem:** agentic 工作流让计算密集的 prefilling 阶段成为 long-context、多轮推理的瓶颈。
- **Method:** 阶段感知量化（phase-aware quantization）——prefilling 用 NVFP4、decoding 保 BF16 精度;硬件高效执行。

#### 🟢 [2605.19660] OScaR — 极端 KV cache 量化的「奥卡姆剃刀」
- **Problem:** 逐通道（per-channel）KV 量化在 long-context 推理中遭遇「Token Norm Imbalance」而失效。
- **Method:** Canalized Rotation + Omni-Token Scaling,配自定义 CUDA kernel 实现 INT2 KV cache,与 FlashDecoding 集成。

#### 🟢 [2605.21226] OCTOPUS — 用八面体参数化优化 KV cache
- **Problem:** KV cache 在 long-context 自回归推理中主导显存带宽与占用。
- **Method:** 在最优平方误差量化下做八面体参数化（octahedral parametrization）——一种延伸 TurboQuant / PolarQuant 的旋转预条件编解码器。
- **Key result:** abstract 未报告量化指标。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21226) · [HF](https://huggingface.co/papers/2605.21226)
