### TL;DR — 本日 research 风向

### Agent frameworks & self-evolution

### 🟢 [2605.22177] Maestro — RL 编排分层「模型-技能」集成
- **Problem:** 单体 LLM + 固定逻辑无法发挥不同模型在不同领域的互补优势。
- **Method:** 用 outcome-based RL 训练轻量 policy，在「模型-技能」分层注册表上做序列决策——每步决定调哪个专家、选哪对 model-skill、何时终止。
- **Key result:** 仅 4B orchestrator 平均准确率 70.1%，超过 GPT-5（69.3%）与 Gemini-2.5-Pro（68.7%）；策略可零样本泛化到未见模型/技能。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22177) · [HF](https://huggingface.co/papers/2605.22177) · [GitHub](https://github.com/jinyangwu/Maestro) ⭐5

### 🟡 [2605.21605] GenEvolve — 工具编排式视觉经验蒸馏的自进化图像生成 agent
- **Problem:** 现有 agentic 生成方法只靠 image-level 标量奖励，反馈太稀疏，难以自我改进。
- **Method:** 把每次生成建模为工具编排轨迹；对同一请求比较多条轨迹，把 best-worst 差异抽象成结构化「视觉经验」，经 on-policy 自蒸馏给出 token 级密集监督。
- **Key result:** 在公开 benchmark 与 GenEvolve-Bench 上较强基线大幅提升，达到当前图像生成框架的 SOTA（abstract 未给具体数值）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21605) · [HF](https://huggingface.co/papers/2605.21605) · [GitHub](https://github.com/MeiGen-AI/GenEvolve) ⭐5

### Agent reliability, evaluation & safety

### 🟢 [2605.14678] π-Bench — 评测长流程中的主动型个人助理 agent
- **Problem:** 现有 benchmark 几乎不评测 agent 能否在用户明说前识别并处理隐藏意图。
- **Method:** 构建 100 个多轮任务、5 类用户画像，引入隐藏意图、跨任务依赖、跨会话连续性，联合度量「主动性」与「任务完成度」。
- **Key result:** 主动协助仍很难；「任务完成」与「主动性」明显是两件事；先前交互对后续主动意图解析有实质价值。
- *Links:* [PDF](https://arxiv.org/pdf/2605.14678) · [HF](https://huggingface.co/papers/2605.14678) · [GitHub](https://github.com/Simplified-Reasoning/Pi-Bench) ⭐7

### 🟢 [2605.22535] TerminalWorld — 在真实终端任务上 benchmark agent
- **Problem:** 现有终端 benchmark 由专家手工策划，与开发者真实工作流相关性弱。
- **Method:** 数据引擎从 80,870 条「野生」终端录像反向工程出 1,530 个验证任务（18 类、1,280 条命令），精选 200 个人工复核子集。
- **Key result:** 8 个前沿模型 + 6 个 agent 最高通过率仅 62.5%；与 Terminal-Bench 仅弱相关（Pearson r=0.20）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22535) · [HF](https://huggingface.co/papers/2605.22535) · [GitHub](https://github.com/EuniAI/TerminalWorld) ⭐1

### 🟢 [2605.22642] Spreadsheet-RL — 用 RL 推进 LLM agent 处理真实电子表格任务
- **Problem:** 现有电子表格 agent 靠通用 LLM 上的专门 prompt，难以应付真实的多步工作流。
- **Method:** RL 微调框架 + Spreadsheet Gym（Python 沙箱暴露 Excel 功能 + 工具路由规则），自动从论坛采集 start-goal 配对表格。
- **Key result:** Qwen3-4B-Thinking 在 SpreadsheetBench 上 Pass@1 从 12.0% 升到 23.4%，自建 Domain-Spreadsheet 上从 8.4% 升到 17.2%。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22642) · [HF](https://huggingface.co/papers/2605.22642) · [GitHub](https://github.com/Spreadsheet-RL/Spreadsheet-RL) ⭐3

### 🟡 [2605.20176] ClinSeekAgent — 为 agentic 临床推理自动化多模态证据搜集
- **Problem:** 现有工作假设证据已被策划好交给模型，而真实流程要求 agent 主动搜集多源证据。
- **Method:** agentic 框架做动态多模态证据搜集（查知识库、导航 EHR、调影像工具），并把高质量轨迹蒸馏进紧凑开源模型。
- **Key result:** 多模态任务上把 Claude Opus 4.6 从 47.5 提到 62.6（+15.1）；蒸馏出的 ClinSeek-35B-A3B 在 AgentEHR-Bench 上较 baseline +11.9。
- *Links:* [PDF](https://arxiv.org/pdf/2605.20176) · [HF](https://huggingface.co/papers/2605.20176) · [GitHub](https://github.com/UCSC-VLAA/ClinSeekAgent) ⭐3

### Post-training (RL, distillation, merging, calibration)

### 🟢 [2605.21467] DelTA — 面向 RLVR 的判别式 token 信用分配
- **Problem:** RLVR 中 response 级奖励如何转成 token 级概率变化仍不清楚；centroid 易被格式 token 等高频模式主导。
- **Method:** 提出「判别器视角」，估计 token 系数放大「side-specific」梯度方向、压低共享/弱判别方向，重加权自归一化 RLVR surrogate。
- **Key result:** 7 个数学 benchmark 上，对最强同规模基线在 Qwen3-8B/14B-Base 分别 +3.26 / +2.62 平均分。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21467) · [HF](https://huggingface.co/papers/2605.21467) · [GitHub](https://github.com/RUCBM/DelTA) ⭐2

### 🟢 [2605.22074] SCRL — 课程式 RL 让 LLM 推理实现可验证的信用分配
- **Problem:** outcome-based RLVR 在难题上低效——正确 rollout 稀少，失败尝试里的部分进展用不上。
- **Method:** 从参考推理链派生可验证子问题、固定最后一个子问题为原题；做子问题级归一化，把 advantage 分配到对应答案 span。
- **Key result:** 7 个数学 benchmark 上较 GRPO 在 Qwen3-4B/14B-Base 分别 +4.1 / +1.9；AIME/IMO-Bench 上 pass@1 +3.7、pass@64 +4.6。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22074) · [HF](https://huggingface.co/papers/2605.22074)

### 🟢 [2605.10158] uPRM — 无监督的过程奖励模型
- **Problem:** PRM 需要逐步专家标注，成本高、难扩展。
- **Method:** 从 LLM next-token 概率派生评分函数，跨一批推理轨迹联合评估「首个错误步」的候选位置，无需任何人工监督或答案验证。
- **Key result:** 在 ProcessBench 上识别首错步较 LLM-as-Judge 提升至多 15% 绝对准确率；作为 test-time 验证器与监督 PRM 相当。
- *Links:* [PDF](https://arxiv.org/pdf/2605.10158) · [HF](https://huggingface.co/papers/2605.10158)

### 🟢 [2605.21850] ACC — 编译 agent 轨迹用于长上下文训练
- **Problem:** 标准 agent SFT 屏蔽工具响应、只训轮级工具选择，导致散落于多轮的长程证据成为监督盲区。
- **Method:** Agent Context Compilation 把搜索/软件工程/数据库 agent 的轨迹转成长上下文 QA 对，训模型不调工具直接作答，显式化「问题↔证据」依赖。
- **Key result:** ACC 训练的 Qwen3-30B-A3B 在 MRCR 达 68.3（+18.1）、GraphWalks 77.5（+7.6），与 Qwen3-235B-A22B 相当。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21850) · [HF](https://huggingface.co/papers/2605.21850)

### ⚪ [2605.21803] 同架构不同容量 — 优化器诱导的谱缩放律
- **Problem:** scaling law 通常把优化器当成固定细节，忽略它对「表示容量」的影响。
- **Method:** 用 FFN 表示的特征谱（soft/hard spectral-rank）测量优化器如何把 FFN 宽度转成可用谱容量；对比 AdamW 与 Muon。
- **Key result:** 在难学的 rare-token 表示上，AdamW hard-rank 缩放 β=0.44，Muon 达 β=1.02（2.3x 缩放指数提升）；匹配的 loss 不意味匹配的表示结构。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21803) · [HF](https://huggingface.co/papers/2605.21803)

### Inference optimization & compiler

### 🟢 [2605.16928] RTPurbo — 百步训练内把 full attention 转成 sparse
- **Problem:** 长上下文推理受 full attention 二次成本拖累，现有高效替代在效率/训练成本/精度间难两全。
- **Method:** 利用 full-attention LLM 本身的内在稀疏性——只对 retrieval head 保留完整 KV cache，引入 16 维 indexer 做稀疏注意力，动态 top-p 选 token。
- **Key result:** 仅几百训练步，1M 上下文 prefill 提速至多 9.36x、decode 约 2.01x，精度近乎无损。
- *Links:* [PDF](https://arxiv.org/pdf/2605.16928) · [HF](https://huggingface.co/papers/2605.16928)

### 🟢 [2605.13734] KVServe — 面向解耦式 LLM 服务的服务感知 KV cache 压缩
- **Problem:** 解耦式 LLM 服务（PD 分离、KV 状态解耦）把 KV 变成跨网络/存储的显式负载，成为端到端瓶颈；静态压缩配置常次优。
- **Method:** 把 KV 压缩统一成模块化策略空间，用贝叶斯 profiling 引擎搜出 3D Pareto 候选集，再用服务感知在线控制器（分析模型 + 轻量 bandit）选配置。
- **Key result:** 集成进 vLLM，PD 分离下 JCT 提速至多 9.13x，KV 解耦下 TTFT 降低至多 32.8x。
- *Links:* [PDF](https://arxiv.org/pdf/2605.13734) · [HF](https://huggingface.co/papers/2605.13734) · [GitHub](https://github.com/hpdps-group/KVServe) ⭐7

### 🟡 [2605.22791] Gated DeltaNet-2 — 在线性注意力中解耦「擦除」与「写入」
- **Problem:** delta-rule 线性注意力用单个标量门同时控制 key 侧擦除旧内容与 value 侧写入新内容，两件事被绑死。
- **Method:** 用 channel-wise 擦除门 b_t 与写入门 w_t 分离两个角色（退化时还原成 KDA / Gated DeltaNet），配 chunkwise WY 算法保留高效并行训练。
- **Key result:** 1.3B 参数、100B FineWeb-Edu token 训练下，在语言建模/常识/检索上优于 Mamba-2/3、Gated DeltaNet、KDA；长上下文 RULER 优势最明显。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22791) · [HF](https://huggingface.co/papers/2605.22791) · [GitHub](https://github.com/NVlabs/GatedDeltaNet-2) ⭐19

### Retrieval / RAG / memory

### 🟡 [2605.22718] WorldKV — 用世界检索与压缩实现高效世界记忆
- **Problem:** 自回归视频扩散世界模型中，full KV-cache 保证一致性但破坏实时性，滑窗恢复吞吐却丢长程一致。
- **Method:** training-free 框架：World Retrieval 把被驱逐的 KV 块存到 GPU/CPU，按相机/动作对应选择性取回；World Compression 按 key-key 相似度剪冗余 token。
- **Key result:** 在 Matrix-Game-2.0 与 LingBot-World-Fast 上，约 2x 吞吐下匹配或超过 full-KV 的记忆保真度，无需微调。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22718) · [HF](https://huggingface.co/papers/2605.22718)

### Multimodal, world models, generation

### 🟡 [2605.22012] LatentOmni — 用统一音视觉潜空间推理重思 omni-modal 理解
- **Problem:** 显式文本 CoT 把连续音视觉信号压成离散 token，削弱时间 grounding、把推理推向语言先验。
- **Method:** 在文本推理中交织音视觉潜状态；用 feature-level 监督对齐潜推理状态，Omni-Sync 位置嵌入（OSPE）维持时间一致性。
- **Key result:** 多个音视觉推理 benchmark 上为评测开源模型中最佳，持续优于 Explicit Text CoT 基线（abstract 未给具体数值）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.22012) · [HF](https://huggingface.co/papers/2605.22012) · [GitHub](https://github.com/yfanDai/LatentOmni) ⭐5

### 🟡 [2605.18577] OmniPro — omni-proactive 流式视频理解的综合 benchmark
- **Problem:** 现有流式视频 benchmark 偏视觉、用轮询/固定时间戳而非真正主动评测、任务覆盖窄。
- **Method:** 2,700 个人工核验样本，9 子任务 3 认知层级；双模式协议——Probe 模式查触发前后的内容理解，Online 模式评测模型自主决定何时响应。
- **Key result:** 11 个模型评测显示：音频带来一致增益但利用率差异大；性能随时间显著下降；非语音音频感知最弱。
- *Links:* [PDF](https://arxiv.org/pdf/2605.18577) · [HF](https://huggingface.co/papers/2605.18577) · [GitHub](https://github.com/RuixiangZhao/OmniPro) ⭐5
