## Part 2 — Research Radar（HuggingFace Papers）

### TL;DR — 本日 research 风向
今日 HF 榜单被 text-to-image / diffusion / 多模态生成主导（Lens、PiD、RankE、Geo-Align 等占半壁），但 Agents 方向出现两条强信号：**agent skills 的自演化与系统化研究**（SkillOpt、Model-Generated Agent Skills）正从"手写 skill"转向"把 skill 当可优化外部状态"。另有 long-horizon agent 训练的信用分配（HinT-SD）与 LLM 容量的 Shannon 视角（理论侧）。按 interest.md，仅 LLM/Agents 方向计入；生成/多模态视觉论文不计入。

### Agent frameworks & self-evolution

#### 🟢 [2605.23904] SkillOpt: Executive Strategy for Self-Evolving Agent Skills
- **Problem:** 闭源模型无法改权重、开源改权重昂贵；手写或一次性 skill 太脆弱，无法持续适配领域。
- **Method:** 把 skill 文档当外部状态，用另一前沿模型做 optimizer，引入训练式控制（rollout 批次取证、文本"学习率"、held-out 验证门、add/delete/replace 编辑）迭代优化 skill。
- **Key result:** GPT-5.5 上 6 个 benchmark 平均 +23.5 分（如 SpreadsheetBench 41.8→80.7）；52 个评测格全部最优或并列；Codex harness +24.8、Claude Code harness +19.1；产物仅 300–2,000 token。
- *Links:* [PDF](https://arxiv.org/pdf/2605.23904) · [HF](https://huggingface.co/papers/2605.23904)

#### 🟢 [2605.23899] From Raw Experience to Skill Consumption: Model-Generated Agent Skills
- **Problem:** 手写 domain-level skill 劳动密集，跟不上 agent 能力与部署范围的扩张。
- **Method:** 系统研究模型自动从过往轨迹蒸馏可复用 skill 工件的全链路（生成→筛选→消费），来自 Fudan / Microsoft Research / SJTU。
- **Key result:** No quantitative result reported（abstract 为系统性研究框架，正文含实验，摘要未给单一汇总 metric）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.23899) · [HF](https://huggingface.co/papers/2605.23899)

### Agent reliability, evaluation & safety

#### 🟢 [2605.21856] The Illusion of Reasoning: 通过 Zero-CoT 截断揭露规避式数据污染
- **Problem:** evals 被改写/扰动后混入训练集（规避式污染），现有检测失效，制造"会推理"的假象。
- **Method:** Zero-CoT Probe (ZCP) —— 黑盒检测：截断 CoT 强制直出答案以暴露记忆捷径，对照同构扰动数据集，用贝叶斯框架算"Contamination Confidence"。
- **Key result:** 污染 benchmark（GSM8K/MATH）上 C_cont≈1.000，干净集（GSM1K）≈0.500 无误报；微调注入污染后污染分支 C_cont≈1.000、held-out 稳定。
- *Links:* [PDF](https://arxiv.org/pdf/2605.21856) · [HF](https://huggingface.co/papers/2605.21856)

### Post-training (RL, distillation, calibration)

#### 🟡 [2605.17873] HinT-SD: 面向 long-horizon agent 的定向后见自蒸馏
- **Problem:** 长时程 agent RL 奖励稀疏，只知成败、不知哪一步出错该如何纠正；逐 turn 反馈又低效。
- **Method:** 用全轨迹后见分析挑出"失败相关"的少数 action span，只在这些 span 上做 feedback-conditioned 自蒸馏（教师 = 同策略 + EMA）。
- **Key result:** BFCL v3 Avg@4 31.56→41.88、AppWorld Avg@4 9.74→18.46；比 dense per-turn 基线最高 +18.80%，且单步训练时间低 2.26×、显存低 1.48×。
- *Links:* [PDF](https://arxiv.org/pdf/2605.17873) · [HF](https://huggingface.co/papers/2605.17873)

### Other (specialized but kept)

#### ⚪ [2605.23901] LLMs as Noisy Channels: Shannon 视角的模型容量与 scaling law
- **Problem:** 单调幂律 scaling law 解释不了 catastrophic overtraining、量化退化等非单调现象（算力增加反而变差）。
- **Method:** Shannon Scaling Law —— 把训练建模成噪声信道传输（参数↔带宽、token↔信号功率），由 Shannon–Hartley 定理推出 LLM 存在"Shannon 容量"上限与 U 型退化。
- **Key result:** Pythia/OLMo2 上一致优于经典与扰动感知 scaling law；用 ≤6.9B/≤180B token 拟合可外推预测未见的 12B/307B token 模型，pooled R²=0.847，单调基线崩溃。
- *Links:* [PDF](https://arxiv.org/pdf/2605.23901) · [HF](https://huggingface.co/papers/2605.23901)

> 非 Agents/LLM 方向（Lens / PiD / RankE / Geo-Align text-to-image，StepAudio 音频，SCOPE FPS world model，PhotoFlow 3D，VGenST 视频，SciAtlas 科研图谱等）按图谱不计入。
