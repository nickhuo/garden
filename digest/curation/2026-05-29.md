## Part 2 — Research Radar（HuggingFace Papers）

### TL;DR — 本日 research 风向
今日 HF 主线压在 **agent 的可靠性与评估**上：异步工具调用、agent 安全对齐、RL 后训练的数据污染检测同日上榜，三者都在补"agent 跑起来之后怎么信它"的洞。第二条线是**激活空间 / LoRA 的可控后训练**（steering、记忆律），与你 wiki 里的 Persona Vectors / On-Policy Distillation 直接接得上。检索方向偏"机制解释"而非新 SOTA。

### Agent 可靠性、评估与安全
### 🟢 [2605.27995] AsyncTool — 异步函数调用评估
- **Problem:** 现有工具调用评测忽略响应延迟与多任务并发，脱离真实场景。
- **Method:** AsyncTool benchmark，模拟工具延迟 + 多异构任务并发，step/sub-task/task 三级评分 + 效率指标。
- **Key result:** 延迟反馈让当前 agent 出现明显性能下降；擅长任务切换与依赖追踪的模型表现更强（abstract 未给单一总分）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.27995) · [HF](https://huggingface.co/papers/2605.27995)

### 🟢 [2605.29801] AgentDoG 1.5 — 轻量 agent 安全对齐框架
- **Problem:** agent 跨环境执行带来广泛安全风险，现有对齐框架难以落地部署。
- **Method:** 分类法驱动的数据引擎 + 影响函数净化 + 训练无关的在线 guardrail（AgentDoG 1.5 模型）。
- **Key result:** 4B 模型在 R-Judge 达 **92.2%**、ATBench **72.4%**，仅用 ~1k 训练样本即逼近 GPT-5.4。
- *Links:* [PDF](https://arxiv.org/pdf/2605.29801) · [HF](https://huggingface.co/papers/2605.29801)

### 🟢 [2605.29888] LaRA — RL 后训练的数据污染检测
- **Problem:** RL 后训练的数据污染少有人查，且输出级（likelihood/entropy）方法对 RL 模型失效。
- **Method:** 逐层表征分析，三个互补指标（扰动敏感度、方向坍缩、局部刚性）聚合成检测协议。
- **Key result:** 在 RL 推理模型上优于现有输出级 baseline（abstract 未给单一数值）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.29888) · [HF](https://huggingface.co/papers/2605.29888)

### 🟡 [2605.30219] 模型何时该改主意 — LLM 上下文信念管理
- **Problem:** LLM 面对新证据时不知何时该更新已有信念，过顽固或过摇摆。
- **Method:** 上下文信念管理框架（contextual belief management）。
- **Key result:** 未在摘要中报告具体指标。
- *Links:* [PDF](https://arxiv.org/pdf/2605.30219) · [HF](https://huggingface.co/papers/2605.30219)

### Agent frameworks & self-evolution
### 🟢 [2605.28424] Skill0.5 — agentic RL 的 skill 内化与利用
- **Problem:** skill-based RL 在"全外化（context 开销大）"与"全内化（过拟合/冲突）"间被迫二选一。
- **Method:** 难度感知 router 分流——通用 skill 走特权蒸馏内化，任务 skill 在简单任务上诊断探针防走捷径。
- **Key result:** ALFWorld + WebShop 上 ID 与 OOD 均超 memory-based / skill-based RL baseline（[GitHub](https://github.com/JasonZhujp/Skill0_5) 开源）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.28424) · [HF](https://huggingface.co/papers/2605.28424) · [GitHub](https://github.com/JasonZhujp/Skill0_5)

### 🟡 [2605.26029] CausaLab — 面向 AI 科学家的交互式因果发现环境
- **Problem:** 缺少可扩展的交互式环境来训练/评测 agent 的因果发现能力。
- **Method:** 可扩展的交互式因果发现 environment（agent-as-scientist）。

### Post-training（RL / 蒸馏 / steering / 校准）
### 🟢 [2605.30076] UniSteer — 激活空间的文本引导 flow matching
- **Problem:** 现有激活 steering 依赖固定方向或任务专用模块，难做细粒度与组合约束。
- **Method:** UniSteer 学一个以自然语言为条件的"激活残差流"速度场，推理时 flow inversion 后注回冻结 LLM。
- **Key result:** 三个目标 LLM 上统一支持行为控制 / 真实性 steering / 细粒度概念 / 多约束指令 / 激活空间分类（abstract 未给单一数值）。
- *Links:* [PDF](https://arxiv.org/pdf/2605.30076) · [HF](https://huggingface.co/papers/2605.30076)
