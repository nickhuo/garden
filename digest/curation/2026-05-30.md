### Code / web agents
### 🟢 [2605.29559] LiteCoder-Terminal — 合成终端环境训练语言 agent
- **Problem:** 训 terminal agent 依赖抓取的外部仓库，域多样性、可控性、缺陷靶向都受限。
- **Method:** LiteCoder-Terminal-Gen——零依赖合成管线，从域规范直接生成可执行可验证的终端环境（11,255 SFT 轨迹 + 602 RL 环境 + DMPO）。
- **Key result:** 32B 变体在 Terminal Bench 1.0/2.0/Pro 达 **29.06% / 18.54% / 34.00% pass@1**，显著超 base。
- *Links:* [PDF](https://arxiv.org/pdf/2605.29559) · [HF](https://huggingface.co/papers/2605.29559)

### Agent frameworks & self-evolution
### 🟢 [2605.29861] Ptah — 可验证多模态 deep research 的多 agent harness
- **Problem:** 多模态 deep research 无确定 ground truth，且要图文交织、可核查。
- **Method:** Ptah——planning/research/writing 分阶段专职 agent + Visual Working Memory + **verifier agent 作为 harness 的 acceptance function**（强制事实接地、引用一致、跨模态一致）。

### Post-training（RL / reward / 蒸馏）
### 🟢 [2605.29648] CorVer — 把可验证奖励推到数学/代码之外（事实 QA）
- **Problem:** LLM 在罕见实体事实上易幻觉；现有句级 reward 依赖昂贵神经 verifier，与 policy 共享盲区。
- **Method:** CorVer——语料接地的轻量奖励：抽主宾对 → Infini-gram 查 Wikipedia 共现统计 → 经句对齐映射成 token 级 advantage。
- **Key result:** TriviaQA 平均 **+4.1pp**；20 个模型×基准格中 **18 格**超 4 个神经 verifier baseline，训练快 **4.8–8.4×**。
- *Links:* [PDF](https://arxiv.org/pdf/2605.29648) · [HF](https://huggingface.co/papers/2605.29648)

### 🟢 [2605.29156] RUBRIC-ARROW — 非可验证域的 rubric reward modeling
- **Problem:** 主观/非可验证场景下 pointwise 打分难，rubric 法因硬布尔聚合产生大量平分。
- **Method:** 交替训练 rubric 生成器与 rubric-conditioned judge，用概率打分 + 分阶段偏好奖励的 alternating GRPO。
- **Key result:** 未在摘要中报告具体指标。
- *Links:* [PDF](https://arxiv.org/pdf/2605.29156) · [HF](https://huggingface.co/papers/2605.29156)

- **Method:** 按置信度驱逐 KV + 混合精度存储（`KV-cache eviction` / `mixed-precision`）。
- **Key result:** 未在摘要中报告具体指标。
