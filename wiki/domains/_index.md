# Domains

Top-level topic areas. Each domain has its own page describing scope, boundaries, and key claims, plus a tag (`tags: [<domain>]`) that filters wiki pages.

## Creation policy

Don't create a `domains/<Topic>.md` until at least 5 wiki pages tag the topic. Until then, the domain lives as a tag only. This prevents domain-page proliferation when a topic is mentioned once and abandoned.

## Cross-domain pages

If a claim genuinely spans two domains, file it in the primary domain and create a `comparisons/<Title>.md` page for the synthesis. Don't create a domain page for the intersection.
